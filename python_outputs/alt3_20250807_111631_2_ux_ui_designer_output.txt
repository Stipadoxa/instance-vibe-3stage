**DESIGN RATIONALE:**

1. **Component Inventory Verification:** All component IDs referenced below have been verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout uses a vertical scrolling structure.  The top section showcases product images using a horizontal scrollable container. This is followed by sections for title, price, condition, seller profile (including a workaround for the star rating), description, and finally, a contact button.  Consistent padding (16px horizontal, 20px vertical for main sections) is used to ensure visual breathing room.  Sections are separated by dividers for clarity.

3. **User Flow Reasoning:** The user flow prioritizes a top-to-bottom information architecture, leading with visually engaging product images, followed by key details (title, price, condition), seller information, then the full product description and, lastly, the call to action ("Contact Seller").

4. **Component Choices:** The `product card` component (147:3743) would ideally be used to structure this, but it is incomplete in the design system (no image gallery functionality). I opted for native elements and component combinations for sections to meet the design goal.

5. **Component Gaps:** The design system lacks a dedicated image gallery component and a star rating component.  For the image gallery, a horizontal `layoutContainer` with image placeholders (`native-rectangle` with image fills) was implemented.  For the star rating, I utilized five instances of the `icon/star` component (635:4752), using variants to represent full, half, and empty stars.

6. **Visual Hierarchy:** Visual hierarchy is established through: 
    - **Size:** Larger text sizes for the title and price to make them prominent.
    - **Color:** Primary colors are used for key information (price, title).
    - **Spacing:** Generous spacing between sections enhances readability and visual separation.
    - **Typography:** Headline and Title styles are used for improved hierarchy.

7. **Variant Selection Rationale:**  The `Button` component (10:3907) uses the "Filled", "Medium", "Default", and "Green" variants for the "Contact Seller" button to provide a clear and prominent call to action. Other variants were avoided to keep design consistent and to prevent unnecessary visual complexity.

8. **Native Element Justification:** Native elements (`native-rectangle`, `native-text`) are used for image placeholders and text elements, particularly for the product description, where the system lacks a rich text component to allow for varied lengths of text.
