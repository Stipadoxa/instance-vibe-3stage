# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish a clear visual hierarchy.  `Headline/Large` is used for the main product title, providing prominence. Section headers (e.g., "Seller Profile," "Specifications") utilize `Title/Large`, while subsection headers within those sections employ `Title/Medium`. Body text uses `Body/Large` for the main product description and `Body/Medium` for specifications, ensuring readability and clear distinction between different levels of information.

- **Spacing System**:  The `itemSpacing` token is used consistently throughout the layout to create visual harmony and separation between elements.  `itemSpacing: 16` is used between major content blocks (e.g., image carousel and description), while `itemSpacing: 8` is used for closer-related elements within the specifications section. Padding is applied to the content container (`paddingTop/Bottom: 16`, `paddingLeft/Right: 16`) to provide breathing room around the content while maintaining edge-to-edge navigation bars.

- **Component Behavior**: The `appbar` component is used for the top navigation, incorporating a back arrow (`leading-icon: "arrow"`) and a share icon (`trailing-icon: "share"`).  The bottom navigation uses the `Bottom navigation` component, displaying icons for home, search, messages, and profile.  The "Contact Seller" button leverages the `Button` component with the `Size: "Large"` and `Type: "Filled"` variants for prominence. The "Report Listing" button utilizes the same component but with `Size: "Medium"` variant for a more subtle appearance.  

- **Color Application**: The color palette is meticulously chosen to align with the design system.  The main background is "White/white100", providing a clean canvas. The primary action button ("Contact Seller") uses "Primary/primary50" for its fill color.  Secondary text uses "Neutral/neutral50".  The price is highlighted using "Primary/primary80" to draw the eye.

- **Layout Strategy**: A vertical layout is used for the overall screen structure, following the three-container mobile app architecture pattern (top nav, content, bottom nav).  This ensures a clear and consistent layout.


## Domain Context Integration

- **Industry Patterns**: The design follows established e-commerce patterns by prominently displaying the product image, price, condition, and seller information.  A clear call to action ("Contact Seller") encourages engagement.

- **User Mental Models**: The layout and information architecture are designed to align with user expectations for product detail pages.  Key information is presented upfront, while more detailed specifications are organized clearly.

- **Contextual Adaptations**:  The `product card` component (though not used directly due to missing ID) has inspired the overall layout and information blocks. The lack of a suitable 'carousel' component in the design system has necessitated a fallback to native elements.

## Design System Compliance

- **Component Verification**: All componentNodeIds used are verified to exist in the provided `DESIGN_SYSTEM_DATA`.
- **Variant Completeness**:  All variants used are specified fully.
- **Text Property Mapping**: Text content is carefully mapped to appropriate textLayers.
- **Schema Adherence**: All properties and values are consistent with the component schemas.
