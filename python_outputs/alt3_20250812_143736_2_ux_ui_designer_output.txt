## Design Decisions
- **Layout Strategy**: Three-section mobile layout (Header, Content,  Footer (BottomNav if mobile)).  Vertical layout used primarily for content sections, horizontal for smaller elements like icon/label pairs.
- **Component Choices**: Utilized system buttons (10:3907) for consistency and native text for flexible content display.  The missing carousel was addressed using horizontal scrolling within a `layoutContainer`.
- **Hierarchy Decisions**: Primary actions are prominently positioned after the product details, while secondary actions are in the header.
- **Gaps Addressed**: No dedicated carousel component exists; implemented using native rectangles and horizontal scrolling.
