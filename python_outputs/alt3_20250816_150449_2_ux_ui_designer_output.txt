# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy:** The design leverages a clear typography hierarchy to guide the user's attention.  "Headline/Large" is used for the main product title, providing prominence. "Title/Medium" styles section headers (e.g., "Seller Profile," "Specifications"), while "Body/Large" and "Body/Medium" are used for the product description and specifications, respectively.  "Caption/Small" is used for the timestamp. This ensures readability and clear visual hierarchy.

- **Spacing System:**  The spacing system uses consistent values from the design system (assumed, as none was provided) to create visual harmony and readability.  `itemSpacing: 16` is used between major sections, while `itemSpacing: 8` is used within smaller groups of related information (e.g., within the seller profile).  Padding values (`paddingTop`, `paddingBottom`, `paddingLeft`, `paddingRight`) are used consistently to ensure sufficient white space around content, enhancing readability and visual breathing room.

- **Component Behavior:** Since a design system is not provided, native components are used to build the interface, simulating the behavior of potential components.  The approach follows a mobile-first strategy, ensuring a smooth and intuitive experience on smaller screens.  All interactive elements (buttons) are assumed to meet minimum touch target sizes.

- **Color Application:**  In the absence of a provided design system, a simulated color palette is used for demonstration purposes only.  The color choices are intended to provide visual clarity and contrast, ensuring readability and creating a cohesive visual experience.  Actual color names would be replaced with real values from the design system.

- **Layout Strategy:** The layout follows a three-container mobile app architecture:  a top navigation bar (containing a back button), a main content area, and a bottom navigation bar. This structure is standard for mobile apps and ensures consistent placement of navigational elements.  The content area uses vertical layout for a natural reading flow, with horizontal layouts employed for elements like the specification table.

## Domain Context Integration

- **Industry Patterns:** The design follows common e-commerce patterns, including prominent product images, clear pricing information, seller profiles with ratings, detailed product descriptions, and clear call-to-actions (Contact Seller, Add to Wishlist, Report Listing).

- **User Mental Models:** The layout prioritizes information crucial to a purchase decision (images, price, condition) at the top.  Secondary information (seller profile, specifications) is placed below, respecting user scanning patterns and mental models of product detail pages.

- **Contextual Adaptations:** The design adapts to the specific context of a used laptop listing.  The "Condition" section includes information relevant to used goods (minor scratches, battery health), providing contextually relevant information.
