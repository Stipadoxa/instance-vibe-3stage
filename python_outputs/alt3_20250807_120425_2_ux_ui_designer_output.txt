## DESIGN RATIONALE

1. **Component Inventory Verification:**  All component IDs referenced below have been manually verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout uses a vertical structure for mobile-first responsiveness.  The top section features a horizontal layout for the quick stats cards, followed by a vertical layout for the activity feed.  Action buttons are placed below the feed for clear visual separation. Bottom navigation is positioned at the very bottom of the screen.  Consistent padding (16px horizontal, 20px top/bottom) is used in content sections for visual breathing room.

3. **User Flow Reasoning:**  The user should quickly grasp their key metrics (listings, views, messages) at the top, followed by recent activity for context. Prominent action buttons encourage engagement with core seller functions (creating new listings and managing messages). Bottom navigation provides persistent access to other dashboard sections.

4. **Component Choices:** I chose the `product card` (147:3743) component for the stats cards because it provides a visually appealing and consistent container for numerical data.  While a dedicated "stats card" component isn't present, this is the closest and most suitable alternative given the available components.

5. **Component Gaps:** The design system lacks dedicated components for an activity feed and a bottom navigation bar.
    * **Activity Feed Workaround:** I will use a `layoutContainer` with vertical layout mode to house individual list items.  Each list item will represent a single activity entry.
    * **Bottom Navigation Workaround:** I will use the `Bottom navigation` (10:4724) component.

6. **Visual Hierarchy:**  Quick stats cards are given prominence through their larger size and placement at the top.  The activity feed is secondary, and the action buttons are clearly emphasized.

7. **Variant Selection Rationale:** For the `Button` (10:3907) components ("Create New Listing" and "View All Messages"), I selected the `Size: Medium`, `Type: Filled`, `Color: Green`, and `State: Default` variants to provide clear, consistent call-to-actions.

8. **Native Element Justification:** No native elements are needed because the available components, with minor layout adjustments using `layoutContainers`, are sufficient to meet all functional requirements.
