# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: Headline/Large is used for the main product title to ensure prominence.  Headline/Medium emphasizes the price and section headers (Specifications, Seller Profile). Body/Medium provides the bulk of the descriptive text, ensuring readability. Caption/Small is used for less important information like "Listed 2 days ago" and the member since date.

- **Spacing System**:  The root container uses `itemSpacing: 0` to ensure the app bar and bottom navigation bar are flush against the screen edges.  `itemSpacing: 8` and `itemSpacing: 16` are used within nested containers to create visual separation between related content blocks. The content container uses standard padding of 16px on all sides.

- **Component Behavior**: The `10:7816` (IMAGES) component is used for the image carousel, leveraging its ability to handle multiple images.  The "Button" component (`10:3907`) is used for all buttons, with variants adjusted based on the action's importance (primary vs. secondary). Visibility overrides are used to show/hide the back arrow icon and hide unnecessary trailing icons in the app bar.

- **Color Application**:  `Primary/primary50` is used for the price to highlight its importance.  `Neutral/neutral90` is used for main headings and `Neutral/neutral50` for the body text.  Consistent color usage enhances visual clarity and aligns with the design system.

- **Layout Strategy**: A vertical layout is used for the main content flow, with horizontal layouts used for grouping secondary actions. This matches user expectations for product detail pages.


## Domain Context Integration

- **Industry Patterns**: The design follows standard e-commerce patterns by prominently displaying product images, price, condition, specifications, seller information, and clear call-to-actions.

- **User Mental Models**: The layout prioritizes key information (images, price, condition) first, followed by supporting details (specs, description, seller profile). This reflects the user's likely scanning behavior when evaluating a product listing.

- **Contextual Adaptations**: The app bar title is kept concise ("Product Details") for navigation context. The full product title is positioned at the start of the content area to avoid app bar length constraints.

- **Component Verification**: All componentNodeIds have been verified against the provided `DESIGN_SYSTEM_DATA`.

- **Variant Completeness**: All components used have their variants appropriately specified.
