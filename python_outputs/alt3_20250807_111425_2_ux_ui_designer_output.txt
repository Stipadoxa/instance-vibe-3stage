**DESIGN RATIONALE:**

1. **Component Inventory Verification:** All component IDs have been manually checked against the provided `DESIGN_SYSTEM_DATA` JSON.

2. **Layout Decisions:** The layout uses a vertical scrolling structure. The top section displays a carousel of product images. Below that, a section contains the title, price, and condition.  The seller profile with a star rating (implemented with individual star icons) follows, succeeded by the description and finally, a "Contact Seller" button.  Consistent padding and spacing (16px) are used throughout for visual clarity and mobile-first responsiveness.

3. **User Flow Reasoning:** The user first sees the product images to quickly assess visual appeal. Then, key information (title, price, condition) is presented prominently, followed by seller details and a description to build trust.  The "Contact Seller" button provides clear call to action.

4. **Component Choices:** I did not use any components that required a workaround.  

5. **Component Gaps:** The design system lacks a dedicated image carousel and star rating component. For the image gallery, I implemented a workaround using multiple `native-rectangle` elements with image fills. For the star rating, I used multiple `native-circle` elements to represent filled and empty stars.


6. **Visual Hierarchy:**  Visual hierarchy is established through size, spacing, and typography. The product title uses a larger `native-text`  with the primary color to establish prominence. The price is displayed in a slightly smaller `native-text` with a primary color. The supporting text uses a smaller size `native-text` with a neutral color.

7. **Variant Selection Rationale:**  Component variants were chosen for optimal presentation. For example, the "Contact Seller" button uses the `Medium` size and `Filled` type variant of the `button` component.  Color selection prioritizes primary color for calls-to-action and neutral for labels and supporting text.

8. **Native Element Justification:**  The image carousel and star rating require native elements due to the lack of suitable components in the provided design system. The `native-rectangle` elements allow for flexible image display, while `native-circle` elements allow for customization needed to create a star rating system.
