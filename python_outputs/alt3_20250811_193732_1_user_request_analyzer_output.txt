**User Request:**  "Design the mobile app screen showing my upcoming workout schedule."


**STAGE 1 ANALYSIS:**
This is a Wellness & Fitness request for a mobile platform. The user wants to view their upcoming workout schedule within a fitness app.


**STAGE 2 CORE CONTENT:**

*Screen Purpose*:  Display the user's upcoming workout schedule, allowing them to view details and manage their training plan.

*Primary Content*:
- Heading: "Upcoming Workouts"
- Key Information:  A list of upcoming workouts, each showing date, time, workout type (e.g., Yoga, HIIT, Strength Training), duration, and location (if applicable).
- Primary Actions: "View Workout Details", "Mark as Complete" (appears only after the workout has started and before it's completed).

*Supporting Elements*:
- Secondary Actions:  "Reschedule Workout," "Cancel Workout" (contextual actions available only before the workout start time).
- Status Indicators: A progress bar indicating percentage of weekly workout goals completed, the current date, and a simple indication of total workout time this week (e.g., "2h 30min of 4h planned").
- Domain Elements:  Integration with wearable device data (if applicable) â€“ a summary showing steps taken before the workout.

*Content Examples*:
- Sample Data:
    -  Monday, October 23rd, 7:00 AM - Yoga (60 min) - Studio A
    -  Tuesday, October 24th, 6:00 PM - HIIT (45 min) - Home
    -  Thursday, October 26th, 12:00 PM - Strength Training (30 min) - Gym B
- Microcopy:  "Tap for workout details," "Mark this workout as complete when finished".
- Empty State: "No workouts scheduled. Add a workout to your plan."


**STAGE 3 NAVIGATION:**

- Top navigation:  Profile, Calendar, Workouts, Progress
- Bottom navigation:  Home, Workouts, Progress, Nutrition, More (Settings etc.)
