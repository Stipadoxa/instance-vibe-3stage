**DESIGN RATIONALE:**

1. **Component Inventory Verification:**  All component IDs used in the design specification below have been verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The profile page uses a vertical layout (`layoutMode`: "VERTICAL") to naturally organize sections.  Each section (personal info, password, address, order history) is contained within its own vertical layout container for clear separation.  The `itemSpacing` is set to 16 for visual breathing room between sections. Padding is used to create visual separation between the edge of the screen and the content.

3. **User Flow Reasoning:** The user flow is linear, starting with the most important information (name, email, and avatar), followed by editable sections (name, email, password, address) and finally the order history. This prioritizes quick access to essential information.  Save buttons are clearly positioned after each editable section.

4. **Component Choices:** 
    * **Avatar:** I chose `10:7773` (avatar / big) over `10:7798` (avatar / single) and `24:10352` (avatar  / stack) because it supports image fills directly, fitting the requirement of displaying a user's profile picture.
    * **Input Fields:** I chose `10:8492` (text input) for name, email, and address fields over `10:8619` (text area) because text inputs are more suitable for single-line entries of this type.  For the password field, I used the same component `10:8492`, but will leverage the "Type" and "State" variants to make the field secure (hidden input type).
    * **List Item:** I chose component `10:10214` (List item) to display the order history since it provides a structured way to show multiple past orders with detail. This component is better for presenting a list of information compared to using other layout strategies.
    * **Buttons:**  I selected `10:3907` (Button) for save buttons. The "Size" and "Color" variants are used to ensure consistency.

5. **Component Gaps:** The design system lacks a dedicated "order summary" component to display order details. I addressed this gap using a combination of `10:10214` (List item) components, allowing for a clean presentation of order information within the order history section, maintaining a consistent user experience.

6. **Visual Hierarchy:** Visual hierarchy is established through sectioning, typography, and spacing.  Section headings are larger and bolder than the content within them.  Clear visual separation is created using dividers (`10:8459`).

7. **Variant Selection Rationale:** For the buttons, the `Size:Medium` variant was chosen for balance and readability, considering the context within the profile screen.  The `Color:Green` variant was selected for "Save" buttons to clearly indicate the action.  For the text inputs, the `State:enabled` and `Type:Filled` variants ensure that the input fields appear consistently.

8. **Native Element Justification:** No native elements were used beyond the basic JSON structure.
