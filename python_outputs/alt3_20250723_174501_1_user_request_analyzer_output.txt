Domain & Context
Domain: Business/SaaS domain with user account management
User Type: Mobile app users managing their account settings and preferences.
Platform Impact: Mobile users need a concise, easily navigable settings page with clear touch targets and minimal scrolling.  Limited screen real estate necessitates prioritization of settings.

User Requirements
Primary Goals: Access and modify account settings, manage notification preferences, log out of the application.
Secondary Goals: Understand account status (e.g., "Verified ID" badge), access help/information related to specific settings (e.g., "What is it?").
User Constraints: Small screen size, touch-only interaction, potential for interruptions, limited attention span.
Ready for handoff if: All critical settings are present and clearly labeled, navigation is intuitive, and touch targets are appropriately sized.  The visual design is consistent with provided reference image and utilizes existing app components.

Content Requirements
Critical: Personal details (edit), Business details (edit), Access to Cars45 auction (toggle), Change phone number, Change email, Change language, Disable chats (toggle), Disable Feedback (toggle), Manage notifications, Change password, Delete my account permanently, Log out.
Supporting: "Verified ID" badge (with explanation),  status indicators (e.g., "Active").
Contextual:  Error messages (if applicable for any settings changes).


Structural & Component Directives
* Layout:  List view with clear section dividers similar to reference image. Top and bottom app navigation bars are included.
* Components: Uses existing app components for list items, toggles, and buttons.
* Styling: Consistent with the reference image and the existing app's design system.  Includes appropriate use of spacing and visual hierarchy to create a clear and organized experience.

Screen Requirements
Primary Actions: Access/modify individual settings, log out.
Secondary Actions:  Access help/information related to settings.
Screen States: Default state, loading state (during settings changes), success/error states (feedback after settings changes).
Feedback Needs: Clear visual feedback after settings changes (e.g., a checkmark or success message), error messages when input is invalid or changes fail.
