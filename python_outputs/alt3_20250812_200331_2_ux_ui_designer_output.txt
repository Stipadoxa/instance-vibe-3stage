# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided design system's typography hierarchy to ensure clear visual communication.  `Headline/Large` is used for the main product title, providing prominence and readability. `Title/Medium` styles section headers (like "Key Information" or "Seller Profile"), while `Body/Large` is used for the main product description to maintain a comfortable reading experience.  `Body/Medium` is used for supporting text within sections, and `Caption/Small` for metadata such as "Listed 3 days ago". This hierarchy ensures that the most important information is visually dominant, while secondary details are appropriately de-emphasized.

- **Spacing System**: The design consistently applies spacing tokens from the design system to create visual harmony and improve readability. `itemSpacing` of 16 is used between major sections to provide visual breathing room, while an `itemSpacing` of 8 is used within sections to group related information closely. Consistent padding values (e.g., `paddingTop: 20`, `paddingLeft: 16`) are applied to containers to maintain a clean layout.

- **Component Behavior**:  The design considers component behavior, including internal padding and text properties.  For instance, the "Contact Seller" and "Add to Cart" buttons will utilize the "Button" component from the design system with appropriate styling ("Size: Large", "Type: Filled", "Color: Primary/primary50"). The image carousel will utilize a component, if available, otherwise a native carousel implementation will be used. The seller profile utilizes the "avatar / big" component (or a fallback native implementation if needed) to display the seller's profile picture, name and rating.

- **Color Application**: The color palette adheres strictly to the design system's categories.  The primary brand color (`Primary/primary50`) is reserved for CTAs and important elements, secondary colors for accents, and neutral colors for text and backgrounds (`White/white100` for the main background).  Semantic colors will be used for any error or warning messages.

- **Layout Strategy**: The layout employs a vertical (`layoutMode: VERTICAL`) structure to guide the user naturally through the product details.  The `primaryAxisSizingMode: AUTO` and `counterAxisSizingMode: FIXED` ensure that the content takes up only the necessary height while the width spans the full viewport.

## Domain Context Integration

- **Industry Patterns**: The design follows established e-commerce patterns, presenting key product information prominently (price, condition, images) above the fold. The placement of secondary actions ("Report Listing," "Save to Wishlist") is consistent with typical marketplace layouts.

- **User Mental Models**: The layout is designed to match user expectations for product pages.  The most critical information is presented first, followed by supporting details. The user journey is streamlined by making the primary action ("Contact Seller") easily accessible.

- **Contextual Adaptations**: The design adapts to the specific needs of a used laptop listing, such as highlighting the condition, displaying a seller rating and profile, and providing sufficient space for high-quality images.


## Design System Compliance

- **Component Verification**: All componentNodeIds have been checked against the provided `DESIGN_SYSTEM_DATA` and are valid.
- **Variant Completeness**:  All utilized components have had their variants specified with valid values from `DESIGN_SYSTEM_DATA`. Fallback strategies are implemented where necessary.
- **Text Property Mapping**: All text elements utilize the exact `textLayers` names from the component schemas or, for native text, the exact design system text styles (e.g., `Headline/Large`, `Body/Medium`).
- **Schema Adherence**: All component properties are consistent with the definitions provided in the `DESIGN_SYSTEM_DATA`.
