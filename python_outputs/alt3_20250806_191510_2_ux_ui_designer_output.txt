SECTIONS:

navigation: [The bottom navigation bar contains icons and labels for Dashboard, Listings, Messages, and Profile.  It provides persistent access to core app sections, aligning with the Dashboard page type pattern.]

metrics: [This section displays quick stats cards showing Total Listings, Views This Week, and New Messages.  These key metrics provide a high-level overview of the seller's shop performance, a core element of the Dashboard pattern.]

actions: [This section features two primary action buttons: "Create New Listing" and "View All Messages".  These CTAs facilitate key seller tasks, fulfilling the actions section in the Dashboard pattern.]

activity: [This section presents a recent activity feed displaying events like purchases and new messages.  This provides timely updates and keeps the seller informed, fitting within the activity section of the Profile and Dashboard patterns.]


DESIGN RATIONALE:

Component Inventory Verification: I have verified all component references against the provided design system data.

Section Planning: The design follows the Dashboard pattern from the SECTION PATTERNS, prioritizing navigation, metrics, actions, and activity feeds.  The user request explicitly mentions these sections, making this a straightforward mapping.

Layout Decisions: The layout is vertically oriented.  The root container uses `"width": 414`, `"counterAxisSizingMode": "FIXED"`, and no padding.  Level 1 containers (navigation, metrics, actions, activity) use `"horizontalSizing": "FILL"` and 16px padding.  Nested containers within these sections will auto-size (hug content). This adheres strictly to the Hierarchical Container Sizing rules.

User Flow Reasoning: The user flow is designed for efficient overview and action.  The metrics provide immediate insights, followed by recent activity for context, and finally, clear actions for immediate tasks. Bottom navigation provides consistent access to other sections.

Component Choices:
- I chose the `Bottom navigation` (id: `10:4724`) component for the navigation section, as it provides a standard mobile navigation solution. I did not use any other component as the existing one met the requirements.
- For the metrics section, I will use a `native-rectangle` element for each card for flexibility in styling and data placement.
- The `Button` component (id: `10:3907`) was chosen for the action buttons due to its clear visual prominence and ease of interaction.
- I will build a custom list item using `native-text` for the activity feed, as there is no explicit component within the design system.

Component Gaps: The design system lacks a dedicated component for the quick stats cards (metrics).  I have addressed this using `native-rectangle` components, styled with color and text components. No specific component for the activity feed was available, so I will utilize `native-text` elements for this section to render the list.

Visual Hierarchy: Visual hierarchy is established through size, contrast, and positioning.  Metrics cards are prominently displayed at the top with larger font sizes. Actions are also highly visible. Activity feed is of a secondary importance hence smaller font size.

Variant Selection Rationale: 
- For the `Bottom navigation` component, I will use the "Mobile" variant.
- For the `Button` component, "Medium", "Filled", and "Green" color variant will be used for the "Create New Listing" button and the appropriate text style from the design system. "Medium", "Filled" and default color will be used for "View All Messages" button. The state will be "Default" for both.

Native Element Justification: Native `native-rectangle` elements are used for the stat cards because no dedicated "stats card" component is available in the Design System.  Native `native-text` elements are used for the activity feed items for the same reason.
