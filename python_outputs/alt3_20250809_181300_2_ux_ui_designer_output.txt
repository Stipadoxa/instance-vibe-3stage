**DESIGN RATIONALE:**

1. **Component Inventory Verification:** All component IDs used in this design have been manually verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout prioritizes a vertical, mobile-first approach.  The header section contains product images and key information (title, price). The body section presents details using a consistent list item structure, followed by the seller information and a call to action.  Spacing between sections and elements is maintained using the defined container padding and itemSpacing properties.

3. **User Flow Reasoning:** The user first encounters the product visuals and pricing, followed by detailed specifications to aid their decision. Seller information builds trust and provides context, leading to the primary action of contacting the seller.

4. **Component Choices:** The `product card` component (147:3743) is used to structure the core product information, including title, price, and attributes. This choice promotes consistency with other product listings.  A `layoutContainer` is used to group multiple components into sections (header, body, actions).

5. **Component Gaps:** The design system lacks a dedicated component for displaying seller ratings.  A workaround using multiple star icon components (635:4752) in a horizontal container has been implemented.

6. **Visual Hierarchy:** Visual hierarchy is established through:
    - Prominent placement of product images and price in the header
    - Clear title and subtitle for high importance
    - Consistent use of secondary text styles for details
    - Clear visual separation of sections
    - Highlighted primary call to action

7. **Variant Selection Rationale:**  For the "Contact Seller" button (10:3907), the "Filled", "Large", "Default", and "Green" variants were chosen to provide a clear and prominent call to action.

8. **Native Element Justification:** `native-text` elements are used to display the product description and seller details which allows customization of the text styles beyond what's directly available in the system's text style components.  `native-rectangle` with image fills are used as placeholders for multiple product images.
