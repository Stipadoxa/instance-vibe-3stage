## STAGE 1: UNDERSTAND THE REQUEST

1. **Domain:** E-commerce
2. **Platform:** Mobile
3. **User Goal:** The user wants to view the details of a used laptop listing on a marketplace app before potentially purchasing it.


## STAGE 2: GENERATE CORE CONTENT

**Main Screen Purpose:** To provide comprehensive information about a specific used laptop listing, allowing the user to make an informed purchase decision and contact the seller.

**Required Information & Actions:**

* **Heading:** 2015 MacBook Pro 13" - Space Gray - Excellent Condition

* **Key Information Blocks:**

    * **Images:** Carousel of at least 4 high-quality photos of the laptop (showing different angles, keyboard, etc.).
    * **Price:** $450
    * **Condition:** Excellent (Minor scratches on the bottom casing)
    * **Seller Profile:**
        * **Profile Picture:** [Image of seller's profile picture]
        * **Seller Name:** John Doe
        * **Seller Rating:** ⭐⭐⭐⭐⭐ (150 Reviews)
    * **Description:** "Lightly used 2015 MacBook Pro 13" in Space Gray.  Excellent condition with minor cosmetic scratches on the bottom.  Runs smoothly, battery health at 85%. Includes original charger.  Perfect for students or professionals."
    * **Specifications:**
        * Processor: 2.7 GHz Intel Core i5
        * RAM: 8GB
        * Storage: 256GB SSD
        * Operating System: macOS Monterey


* **Primary Actions:**

    * **Contact Seller:** Button that initiates a chat or messaging system within the app.

* **Secondary Actions:**

    * **Report Listing:** Button to report suspicious activity.
    * **Add to Favorites:** Button to save the listing for later.


* **Required Status Indicators:**

    * **Listing Updated:** "Updated 2 hours ago" (dynamically updated)


## STAGE 3: ADD NAVIGATION

**Top Navigation:**

* **Back:** Icon (left-aligned)
* **Share:** Icon (right-aligned)

**Bottom Navigation:**

* **Home:** Icon and text label
* **Search:** Icon and text label
* **Messages:** Icon and text label
* **Profile:** Icon and text label

**Note:**  If the app features a "Favorites" section, that could replace "Search" as a bottom navigation item.  The choice is dependent on the overall app structure which isn't specified in the original request.
