# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**:  Headline/Large is used for the main product title to ensure prominence. Headline/Medium highlights the price, making it easily noticeable. - **Spacing System**:  The `itemSpacing` of 8 and 16 within the content area ensures a visually balanced layout.  Padding of 16 is used on the sides of the content area and 20 on the top and bottom to enhance readability and visual separation from the top and bottom navigation.

- **Component Behavior**: The image carousel (`10:7816`) displays multiple product images. The seller profile uses `10:7798` for the avatar. The contact and report buttons (`10:3907`) are styled to emphasize the primary action.

- **Color Application**: Primary/primary50 (main brand green) is used for the price to highlight its importance. Neutral/neutral50 (grey) is applied to the timestamp for a subtle appearance.

- **Layout Strategy**: The three-container mobile architecture is strictly followed (top navigation, content area, and bottom navigation). The overall vertical layout ensures a clear flow of information, while the horizontal alignment of seller info improves readability.

## Domain Context Integration

- **Industry Patterns**: The design follows e-commerce conventions by prominently displaying the product image, price, and seller information. The contact seller button is the primary call to action, aligning with standard purchasing flows.

- **User Mental Models**: The structure reflects the user's expected information flow for such an interface. Users anticipate seeing the main title, price, and images first, then details, before reaching action buttons.

- **Contextual Adaptations**: The `product card` component, if available, would have been utilized; however, it is not currently available in DESIGN_SYSTEM_DATA. The UI mimics the functionality of this component without using it.
