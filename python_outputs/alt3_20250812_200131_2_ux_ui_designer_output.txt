# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish a clear visual hierarchy.  `Headline/Large` is used for the main product title to emphasize its importance.  `Title/Medium` styles section headers (like "Specifications" or "Seller Information"), providing clear visual separation. `Body/Medium` is used for descriptive text to ensure readability, while `Caption/Small` is reserved for less crucial information such as the listing date.

- **Spacing System**:  Consistent spacing is maintained using the design system's `itemSpacing` tokens.  A `itemSpacing` of 16 is used between major sections for better visual breathing room, while an `itemSpacing` of 8 is used within sections for closely related content.  Container padding follows a similar pattern, ensuring consistent spacing around content blocks.

- **Component Behavior**: The design system's `Button` component is used for both primary ("Contact Seller") and secondary actions ("Save Listing," "Report Listing"). Variants are used to distinguish primary actions (Large, Filled, Primary color) from secondary ones (Medium, Outlined, Neutral color).  The image carousel is implemented using the design system's image component, leveraging its built-in features for image enlargement.

- **Color Application**: The color palette adheres strictly to the provided design system.  Primary colors are used for CTAs ("Contact Seller," "Add to Cart"), while neutral colors are used for text and background elements to maintain a clean and consistent visual experience.  Error states, if any, would utilize the semantic "Error" colors from the system.

- **Layout Strategy**: A vertical layout is used to guide users through the information naturally.  The use of `layoutContainer` with `layoutMode: "VERTICAL"` and appropriate `itemSpacing` and padding ensures a clear and organized presentation.


## Domain Context Integration

- **Industry Patterns**: The design follows established e-commerce patterns, presenting key product information prominently (price, condition, specifications) before less critical elements.  User reviews and seller ratings are given appropriate visual prominence.  Clear CTAs are emphasized for a streamlined purchase journey.

- **User Mental Models**: The layout is designed to match user expectations for product pages.  Prominent product images are placed at the top followed by concise, easily scannable details and clear call to action buttons.  This arrangement follows established mental models for users accustomed to navigating e-commerce sites.

- **Contextual Adaptations**: While the design system might not have a dedicated "product detail page" component, the existing components are used creatively to construct the page.  For example, a `layoutContainer` serves as the primary layout structure, while the `Button` and image components handle the interaction and display elements.  If a component is not available, a fallback using native components would be applied according to the specifications.


## Design System Compliance

- **Component Verification**: All componentNodeIds used are verified against the `DESIGN_SYSTEM_DATA` provided.  Fallback strategies are implemented for components not present in the provided dataset.

- **Variant Completeness**: All selected components utilize variants appropriately to match the content and context. For example, buttons are specified with sizes and styles that maintain visual consistency.

- **Text Property Mapping**: Text content is mapped to the correct textLayers within the components, ensuring proper styling.  Where the design system lacks the ideal component, native `text` elements with correctly specified textStyles are utilized.

- **Schema Adherence**:  The JSON structure strictly adheres to the component specifications given within `DESIGN_SYSTEM_DATA`.
