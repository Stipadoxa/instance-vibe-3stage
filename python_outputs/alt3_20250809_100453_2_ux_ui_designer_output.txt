**DESIGN RATIONALE:**

1. **Component Inventory Verification:** All component IDs have been checked against the provided `DESIGN_SYSTEM_DATA`.  `10:3907` (Button), `10:7798` (Avatar), and `147:3743` (Product Card) are used.  No unverified components were used.

2. **Layout Decisions:** The layout uses a vertical structure for a mobile-first approach. The top section features a hero image (using `native-rectangle` with image fill), followed by a title and price.  A horizontal layout (`layoutContainer` with `layoutMode: "HORIZONTAL"`) is used for the seller's profile (avatar and rating stars - using `native-circle` for stars) and the description is in a vertical container.  A `native-text` workaround is used for detailed styling of certain text elements.

3. **User Flow Reasoning:** The user first sees the product image, then title and price â€“ key information. Scrolling reveals additional details (condition, description, seller profile) which encourage further engagement and trust-building.  The prominent "Contact Seller" button at the bottom facilitates direct interaction.

4. **Component Choices:** I chose the `10:3907` Button component for the "Contact Seller" button due to its established visual style and clear affordance within the design system. It's readily recognizable to the user.

5. **Component Gaps:** The design system lacks a dedicated image carousel/gallery component. A workaround is not implemented as the requirements did not specify a full carousel or functionality beyond showing multiple images; thus, multiple `native-rectangle` components with image fills are used to display the images side by side.  A star rating component is also missing; thus, the rating is implemented as `native-circle` components to represent the stars.


6. **Visual Hierarchy:** The product title and price are given prominence through larger font sizes and distinct colors. Important details are displayed clearly in appropriate text styles, which use the system text styles.  Visual separation and grouping of elements is ensured by padding and dividers (`10:8459` Divider) as needed.

7. **Variant Selection Rationale:**  For the "Contact Seller" button (`10:3907`), the "Filled" type and "Green" color are selected to convey a clear call to action.

8. **Native Element Justification:**  `native-rectangle` is used for image placeholders (multiple product photos) as the design system does not provide a proper image gallery component. `native-circle` components are used to simulate the star rating because a dedicated rating component is not available in the design system.  `native-text` is utilized to provide the precise typographic styling not offered by available system text styles.
