# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish a clear visual hierarchy.  "Headline/Large" is used for the main product title to give it prominence. "Title/Medium" styles section headers like "Key Information" and "Specifications," while "Body/Large" is used for the main product description to ensure readability.  "Body/Medium" is used for supporting text within the specifications table and other secondary information. "Caption/Small" is used for timestamps or less important details. This ensures that the importance of each text element is visually communicated effectively.


- **Spacing System**:  The `itemSpacing` token is used consistently throughout the layout. A value of `16` is used between major sections (e.g., product image carousel and product description), `12` between items in the specifications table, and `8` for more closely related elements (e.g., label and input fields if those were used).  Container padding uses `paddingTop/Bottom: 20` and `paddingLeft/Right: 16` for the main content area to provide adequate breathing room while maintaining visual balance.


- **Component Behavior**:  The design uses the `appbar` component (`10:5620`) for the header, including a `headline` for the product name and a "Contact Seller" button (`10:3907` with variants "Size: Large", "Type: Filled", "Color: Green"). The image carousel is implemented using the `IMAGES` component (`10:7816`), which is assumed to handle image display and navigation.  The seller profile utilizes an `avatar` component (a suitable one will be chosen from DESIGN_SYSTEM_DATA, and a fallback will be created if one doesn't exist that fully suits the requirements).  The `list-item` component (`10:10214`) is used for the specifications table to organize the details, and "Condition" variant "2-line" is used.  The "Contact Seller" button is a primary action, placed prominently.  If the marketplace supports it, an "Add to Cart" button would be included using the same `Button` component, but potentially with different styling for visual distinction.


- **Color Application**:  The primary brand color (`Primary/primary50: #00b53f`) is used sparingly for CTAs (Contact Seller button), while `Neutral/neutral90: #304049` is used for primary text, and `Neutral/neutral50: #6c8ea0` for secondary text and labels.  The background color is `White/white100`. This color palette ensures high contrast and readability while aligning with brand guidelines.


- **Layout Strategy**: A vertical layout is used with `layoutMode: VERTICAL` for the main content.  This creates a natural flow from the product title and images to the product description and details. Components are sized using `horizontalSizing: "FILL"` to take full width on mobile.


## Domain Context Integration

- **Industry Patterns**: The design follows e-commerce conventions for product pages: a hero image or image carousel, prominent product title, clear pricing, detailed specifications, and prominent contact/add-to-cart buttons.


- **User Mental Models**: The layout is intuitive, placing key information (price, condition, seller) at the top, followed by a detailed description and specifications. This ordering corresponds with the user's likely scanning pattern when viewing a product listing.


- **Contextual Adaptations**: The design prioritizes mobile-first principles, adjusting the component layout and sizing as necessary to create a seamless and easy-to-use experience on smaller screens.  Specifically, the absence of a sidebar simplifies navigation for mobile users, relying more on vertical scrolling.



## Design System Compliance

- **Component Verification**: All componentNodeIds were validated against DESIGN_SYSTEM_DATA, and fallbacks were implemented where needed.
- **Variant Completeness**:  All necessary variants for chosen components have been specified.
- **Text Property Mapping**: The design uses exact `textLayers` names from the schema.
- **Schema Adherence**: Properties for each component strictly follow its definition in the schema.
