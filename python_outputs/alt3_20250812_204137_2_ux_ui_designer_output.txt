# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish a clear visual hierarchy.  "Headline/Large" is used for the main product title, providing prominence and immediate clarity. Section headers (like "Seller Profile" and "Specifications") utilize "Title/Medium" for a distinct yet less dominant style. Body text, including the product description and specifications, employs "Body/Medium" for readability.  "Label/Medium" is used for labels within the specifications section to further distinguish them from the values.  Finally, "Caption/Small" is used for the timestamp ("Last updated: 2 hours ago"). This consistent application of the typography system ensures clear communication of information importance.


- **Spacing System**: The `itemSpacing` token is used consistently to create visual breathing room and to group related information logically.  A `paddingTop` and `paddingBottom` of 20 are used for main sections, giving the screen a visually balanced feel. Internal padding within components is respected and not overridden unless absolutely necessary to maintain consistency with the design system.


- **Component Behavior**: The design system's `appbar` component (10:5620) is utilized for the top navigation, featuring a back arrow ("arrow" icon from 635:4715). The bottom navigation utilizes the  `Bottom navigation` component (10:4724), displaying icons for Home, Search, Messages, and Profile.  The "Contact Seller" button utilizes the "Button" component (10:3907) with the "Large" size variant for prominence. "Add to Wishlist" and "Report Listing" buttons use the same component but with the "Medium" and "Small" size variants, respectively, reflecting their secondary importance.  The seller's rating is represented using a combination of native text and the "star" component (635:4752), if available, with the correct number of filled stars.  If not, the system falls back on a more basic representation.


- **Color Application**:  The color palette is strictly adhered to.  The primary brand color (`Primary/primary50: #00b53f`) is used sparingly for CTAs (Contact Seller button). Neutral colors (`Neutral/neutral90: #304049` and `Neutral/neutral50: #6c8ea0`) dominate for text, ensuring high contrast and readability. Backgrounds consistently use `White/white100: #ffffff`. This ensures a clean, consistent, and branded experience.


- **Layout Strategy**: A vertical layout (`layoutMode: "VERTICAL"`) is used for the main screen, allowing for a natural top-to-bottom flow of information.  The image carousel and key information blocks are vertically stacked. Within sections, elements like specifications are arranged in a logical vertical structure, while horizontal alignment is used where appropriate (e.g., within the specifications section). The bottom navigation bar remains fixed at the bottom of the screen for easy access to other sections.


## Domain Context Integration

- **Industry Patterns**: The design adheres to e-commerce conventions, showcasing a product image gallery, clear pricing, condition details, seller profile information (including ratings and reviews), and prominent call-to-action buttons.

- **User Mental Models**: The top-to-bottom layout and logical grouping of information cater to the user's mental model of browsing product details. Information is arranged in a way that intuitively guides the user's eye through the key details before prompting them with the contact action.

- **Contextual Adaptations**:  While the design system provides many components, some elements might require custom styling or native elements for optimal presentation. For example, the product image gallery and the seller's profile picture require the `image` component or native elements for image display, given that no suitable image-carousel component is explicitly defined in `DESIGN_SYSTEM_DATA`.


## Design System Compliance

- **Component Verification**:  All componentNodeIds used in the design specification are verified against the provided `DESIGN_SYSTEM_DATA`.

- **Variant Completeness**: All selected components have variants defined and set appropriately to ensure functional design.

- **Text Property Mapping**: All text content is mapped to appropriate textLayers names within the components or native text elements use exact design system style names.

- **Schema Adherence**:  All component properties strictly adhere to the definitions within `DESIGN_SYSTEM_DATA`.
