# DESIGN RATIONALE

## Component Intelligence Applied
- **Typography Hierarchy**:  Headline/Large is used for the main product title, Title/Large for the price, Title/Medium for the "Specifications" header, and Body/Medium for the description and specifications details. Caption/Small is used for the listing date. This hierarchy ensures clear visual prominence of key information.
- **Spacing System**:  The `itemSpacing` of 16px is used consistently throughout the content area to maintain a comfortable visual rhythm between elements.  Padding is used to create breathing room around sections and individual components.
- **Component Behavior**: The image component is used to display the product gallery. The button component is utilized for the "Message Seller" CTA. The heart icon is added for the favorites feature.
- **Color Application**: Primary/primary50 is used for the price to highlight its importance. Neutral colors (neutral90, neutral50) are used for the main text and supporting information to ensure readability.
- **Layout Strategy**: A vertical layout is employed for the main content, allowing for a natural top-to-bottom flow of information.  Horizontal layouts are used where appropriate to group related items (e.g., price and favorites icon).

## Domain Context Integration
- **Industry Patterns**: The design incorporates common e-commerce patterns such as product image carousels, pricing displays, product descriptions, and seller profiles.  User reviews are incorporated to build trust.
- **User Mental Models**: The layout follows a natural progression of information, starting with the product title and image, then moving to price, condition, seller details, and finally specifications and call-to-action.
- **Contextual Adaptations**: A "Message Seller" button is used as the primary action, suitable for the domain context.

## Design System Compliance
- **Component Verification**: All componentNodeIds are verified against the provided DESIGN_SYSTEM_DATA.
