## SECTION 1: DESIGN RATIONALE

# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish a clear visual hierarchy.  "Headline/Large" is used for the main product title to give it prominence. "Title/Medium" styles section headers (like "Seller Profile" and "Specifications"), while "Body/Large" is used for the main product description and "Body/Medium" for supporting details within sections.  "Caption/Small" is used for timestamps and metadata like "Last updated" and "Watchers". This ensures readability and guides the user's attention effectively.

- **Spacing System**: The `itemSpacing` token is used consistently to create visual breathing room between elements.  A value of `16` is used for major section separations (e.g., between the image carousel and the description), while `8` is used for tighter groupings of information within sections.  Padding within containers is also applied consistently using multiples of 8, ensuring a harmonious visual rhythm.  The `Mobile App Architecture Pattern` dictates padding of 0 on the root container, allowing navigation to touch edges.  Content padding is consistent inside its own container.

- **Component Behavior**:  The design utilizes the `appbar` component for the top navigation, incorporating a back arrow (`635:4715` with `style: back`) as the leading icon. The bottom navigation employs the `Bottom navigation` component (`10:4724`) to house Home (`635:4317`), Search (`635:4105`), Messages (`635:4362`), and Profile (`635:4160`) icons.  Buttons use the `Button` component (`10:3907`) with appropriate sizes and types:  "Contact Seller" is a `Size: Large`, `Type: Filled` button, and secondary actions are `Size: Medium`, `Type: Text` buttons.

- **Color Application**:  The color palette adheres strictly to the provided design system. The primary brand color (`Primary/primary50`) is used sparingly for CTAs.  Neutral colors (`Neutral/neutral90`, `Neutral/neutral80`, `Neutral/neutral50`) are used extensively for text to ensure readability against various backgrounds.  Backgrounds utilize `White/white100` for a clean and uncluttered look.

- **Layout Strategy**: A vertical layout (`layoutMode: VERTICAL`) is used for the main screen structure, following the three-container mobile app architecture (top nav, content, bottom nav). This ensures all screen elements are presented clearly within the constraints of a mobile viewport and maintain a consistent layout throughout the app. The `primaryAxisSizingMode: AUTO` on the main content container allows for content to grow naturally, keeping navigation bars fixed.


## Domain Context Integration

- **Industry Patterns**: The design follows established e-commerce patterns for product detail pages, such as a prominent product image carousel, clear pricing, detailed product description, seller information, and prominent call-to-action buttons.

- **User Mental Models**:  The layout mirrors the user's expectations for viewing a product detail page, placing key information (images, price, condition) prominently, and organizing supporting details in a logical, visually separated manner. The collapsible specification section reduces visual clutter while preserving detailed information.

- **Contextual Adaptations**: The design adapts to potential empty states by providing placeholder messages for missing images, reviews, or descriptions. This mitigates user confusion and frustration when information is incomplete.

## Design System Compliance

- **Component Verification**: All componentNodeIds have been checked against `DESIGN_SYSTEM_DATA` for existence and validity.

- **Variant Completeness**: All selected components have their variants specified using valid values from `DESIGN_SYSTEM_DATA`.

- **Text Property Mapping**:  Text content is accurately mapped to the correct text layers within the chosen components and native text elements.

- **Schema Adherence**:  All component properties and attributes adhere precisely to the specifications defined in `DESIGN_SYSTEM_DATA`.
