# DESIGN RATIONALE

## Component Intelligence Applied
- **Typography Hierarchy**:  Headline/Large is used for the product title, Title/Large for the price, Title/Medium for the seller name, Body/Medium for the description and condition, and Body/Small for specifications. Caption/Small is used for the seller rating and last updated time. This hierarchy ensures clear visual distinction between different levels of importance.

- **Spacing System**:  The `itemSpacing` and padding are used consistently throughout the design, creating a visually balanced layout.  Larger spacing is used between major sections, while smaller spacing is used within sections to group related information. The three-container architecture ensures no unnecessary spacing between navigation and content areas.

- **Component Behavior**:  The image carousel uses the `Thumbnail` component to display multiple product images, and the `Button` component is used for primary and secondary actions. The seller profile uses the `avatar / single` and `rating` components for a consistent user experience. The `Badge` component is used for the "Last Updated" timestamp, which is a standard pattern.

- **Color Application**: Primary colors are used for the price and main CTA button to attract attention, while neutral colors are used for the background and text for optimal readability. Semantic colors are avoided for body text. The primary color (#00b53f) is consistently used.

- **Layout Strategy**: The three-container architecture is used for a mobile-first design, ensuring a clear separation between navigation and content. The layout follows a vertical flow for readability, with horizontal layouts used for grouping related elements like the price and last updated timestamp, and the seller profile image and details.

## Domain Context Integration
- **Industry Patterns**: The design incorporates common e-commerce patterns, including prominent product images, pricing, seller profiles, product specifications, and clear call-to-actions.

- **User Mental Models**: The layout is designed to match user expectations for product detail pages, presenting information in a logical and intuitive way.  The most prominent information is displayed first, followed by supplementary details.

- **Contextual Adaptations**: The design system components are adapted for the specific use case of displaying used laptop details, and some are used as fallback options in the absence of a perfect match.

## Design System Compliance
- **Component Verification**: All componentNodeIds have been verified against the provided `DESIGN_SYSTEM_DATA`.
- **Variant Completeness**:  All required variants for the chosen components are specified with valid values from the design system.
- **Text Property Mapping**: Exact textLayers names from the component schema are used where applicable.
- **Schema Adherence**: Properties match the component definitions precisely.
