# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to ensure clear visual hierarchy.  The main product title ("2015 MacBook Pro 13\" - Space Gray - Excellent Condition") uses the `Headline/Large` style for prominence. Section headers within the product details (e.g., "Specifications") will use `Title/Large`, while sub-sections within those (e.g., "Processor") will employ `Body/Medium`. The seller's username and rating are styled with `Body/Medium` and `Title/Small` respectively, creating a clear distinction from the main content.  Body text uses `Body/Medium` for readability.

- **Spacing System**:  The `itemSpacing` token is consistently applied to create visual breathing room between sections.  A value of `16` is used between major sections (e.g., image carousel and product description), while `8` is employed within tighter groups like the seller profile information. Padding values are chosen based on context; larger sections use `paddingTop/Bottom: 20`, while smaller sections might use `8` or `12`.

- **Component Behavior**: The `list-item` component is used for the specifications section with the `Condition` variant set to "2-line" to accommodate the key-value pairs.  This ensures consistent styling and behavior across the app. Buttons are used for actions, with `Contact Seller` as a primary button and `Add to Wishlist` and `Report Listing` as secondary buttons.  The appropriate size and style variants are applied as needed.


- **Color Application**:  The color palette is carefully chosen to maintain consistency and provide clear visual cues.  Backgrounds primarily use `White/white100` for readability. Text uses `Neutral/neutral90` for optimal contrast.  Secondary elements and accents, like the buttons, employ `Primary/primary80`.  The selected color names are consistently referenced as provided in `DESIGN_SYSTEM_DATA` to maintain accuracy.


- **Layout Strategy**: A vertical layout is used to organize content naturally from top to bottom. The main sections (images, price, condition, seller info, description, specifications) are arranged vertically. The bottom navigation bar is a horizontal row for easy access. The `horizontalSizing` property is set to `"FILL"` to ensure full-width usage where appropriate.  `primaryAxisSizingMode` is set to "AUTO" so sections grow to fit their content, and `counterAxisSizingMode` is "FIXED" to leverage the 375px viewport width.


## Domain Context Integration

- **Industry Patterns**: The design follows typical e-commerce conventions for product listings, including prominent product images, price display, seller information (avatar, username, rating), detailed product description, specifications and clear call-to-actions (Contact Seller).

- **User Mental Models**: The layout anticipates the user's expectations of a product detail screen on an e-commerce platform.  Users expect to see images first, followed by key details (price, condition), then the seller’s profile.  The specifications are collapsible to improve initial clarity.

- **Contextual Adaptations**: No significant adaptations were needed as the design system provided sufficient components for this e-commerce use case.

- **Variant Completeness**: All necessary variants are specified with valid values from the design system.
