**DESIGN RATIONALE:**

1. **Component Inventory Verification:** All component IDs have been verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout uses a vertical scrolling approach, common for product detail pages.  The top section features a full-width image gallery followed by product details (title, price, condition), then seller information (profile picture, rating), and finally the product description and a "Contact Seller" button.  Spacing is consistent using the Pattern A and Pattern C container patterns (16px itemSpacing, 16px padding).

3. **User Flow Reasoning:** The user first sees a visually appealing image gallery, then quickly grasps key information (title, price, condition) before delving into more detailed seller information and product description. This prioritizes key purchase drivers. The "Contact Seller" button is strategically positioned for easy access after review.

4. **Component Choices:**  I would ideally use a dedicated image gallery component and a star rating component, but these are missing.  The `product card` component is a good choice to reuse since it contains images, thus being the closest approximation to a gallery.

5. **Component Gaps:** The design system lacks an image gallery and a star rating component. The workaround for the image gallery involves using multiple instances of the `native-rectangle` component with image fills. The star rating is approximated using multiple instances of the `icon/star` component from the design system, manually arranged to visually represent a rating (in place of a rating component).

6. **Visual Hierarchy:** Visual hierarchy is achieved through:
    - Prominent placement of product images at the top.
    - Larger `Headline/Large` style for the product title.
    - `Title/Medium` style for the price, shown prominently beneath the title.
    - `Label/Medium` style for condition details.
    - `Body/Medium` style for the product description.

7. **Variant Selection Rationale:**  For the "Contact Seller" button (`10:3907`), I selected variants `Size: Medium`, `Type: Filled`, `State: Default`, and `Color: Green`.  This prioritizes actionability with a clear visual cue. Other variants were not needed in this context.

8. **Native Element Justification:**  `native-rectangle` components are used for the image gallery because no dedicated image gallery component exists in the provided design system. `native-text` elements are used to create the product description because the provided text components don't offer enough flexibility or control over styling, particularly for the long format product description.
