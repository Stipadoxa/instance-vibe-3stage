## SECTION 1: DESIGN RATIONALE

# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish a clear visual hierarchy.  "Headline/Large" is used for the main product title to emphasize its importance. "Title/Medium" styles section headers (like "Seller Profile" and "Specifications") creating clear visual separation. "Body/Large" is used for the main product description, while "Body/Medium" styles the individual specification items for readability. "Caption/Small" is used for the "Last Updated" timestamp, indicating its secondary importance.

- **Spacing System**:  Consistent spacing is maintained using the design system's `itemSpacing` tokens.  `itemSpacing: 16` is used between major sections (e.g., Images and Price), providing sufficient visual breathing room. Within sections, `itemSpacing: 8` is used for tighter grouping of related items (e.g., within the Specifications section). Container padding (`paddingLeft/Right: 16`, `paddingTop/Bottom: 20`) ensures comfortable margins around content blocks.

- **Component Behavior**: The design uses the `appbar` component for the top navigation, leveraging its built-in text styles and incorporating a back arrow icon (`635:4715`, variant "back")  for navigation.  The `Bottom navigation` component is utilized for the bottom bar, mapping icons to their respective sections (Home: `635:4317`, Search: `635:4105`, Messages: `635:4362`, Profile: `635:4160`).  The `product card` component (fallback to `list-item`) is used to structure the main product information, although several properties needed to be constructed with native elements due to the component's limitations.  The "Contact Seller" and "Add to Favorites" actions are implemented using the `Button` component.

- **Color Application**: The color palette adheres strictly to the design system.  "Primary/primary50" is used for the primary action button ("Contact Seller") to highlight its importance.  Neutral colors are used for text and background, maintaining readability and consistency ("Neutral/neutral90" for headings and primary text, "Neutral/neutral50" for secondary text). The price is highlighted with the primary color.


- **Layout Strategy**: The layout uses a vertical layout (`layoutMode: "VERTICAL"`) to guide the user through the information logically. The `itemSpacing` provides appropriate visual separation between sections.  The image carousel is implemented using a native layout container with horizontal alignment to support image swiping.


## Domain Context Integration

- **Industry Patterns**: The design mirrors e-commerce best practices by prominently displaying high-quality product images, price, condition, and seller information, all placed above the call to action to encourage purchase.

- **User Mental Models**:  The layout naturally flows from top to bottom, mimicking typical product detail pages.  Essential information is placed above the fold (above the screen’s initial view), while less important information appears after.

- **Contextual Adaptations**: The lack of a suitable "product card" component in the design system necessitated a fallback to a `list-item` component, which was adapted by creating several elements with native elements to satisfy the requirements of the UI.



## Design System Compliance

- **Component Verification**: All componentNodeIds have been verified against the provided `DESIGN_SYSTEM_DATA`.
- **Variant Completeness**:  All selected components use valid variants as defined in `DESIGN_SYSTEM_DATA`.
- **Text Property Mapping**:  Text content is mapped to the appropriate `textLayers` properties of the chosen components, or to `native-text` elements when needed.
- **Schema Adherence**:  All component properties adhere to the definitions in `DESIGN_SYSTEM_DATA`.
