# DESIGN RATIONALE

## Component Intelligence Applied
- **Typography Hierarchy**:  Headline/Large is used for the main product title to ensure prominence. Title/Large is used for section headers (Specifications, Description, Shipping Information). Title/Medium is used for seller name and  "Last updated" information. Body/Medium is used for the product description and specifications. Caption/Small is used for less important text like the seller rating.  This hierarchy ensures clear visual prioritization of information.

- **Spacing System**:  The `itemSpacing` of 8 is used within sections to create visual separation between items. A `paddingTop` and `paddingBottom` of 16 is used in the main content container to create appropriate visual breathing space around the content.  This spacing system ensures readability and visual clarity.

- **Component Behavior**: The `10:7816` component is used to display multiple product images.  The "Contact Seller" button utilizes the `10:3907` component, with variants adjusted for size and style consistency.  The "Add to Wishlist" button utilizes the same component, with a smaller size to reflect its secondary nature. The back button is implemented using the `10:5620` appbar component and visibility overrides are used to hide unnecessary trailing icons.  This leverages component properties and variants to create a consistent and user-friendly experience.

- **Color Application**: Primary/primary50 is used for the price to emphasize its importance. Neutral/neutral90 is used for primary text, while Neutral/neutral50 is for secondary text. This color palette enhances readability and aligns with brand guidelines.


- **Layout Strategy**: A vertical layout is used for the overall screen structure, with horizontal layouts used for certain sections like price and update time, and the seller details to effectively utilize screen space.  The `layoutGrow: 1` property on the main content container ensures that the bottom navigation bar remains at the bottom of the screen regardless of the length of content. This ensures a consistent and intuitive user experience across varying screen sizes.

## Domain Context Integration
- **Industry Patterns**:  The design follows standard e-commerce patterns by placing prominent product imagery, pricing, and key specifications early on the screen. Seller information is included to foster trust.  The use of buttons to allow contact with the seller and to add the item to a wishlist are in accordance with common e-commerce user journeys.

- **User Mental Models**: Users expect to see product details and imagery prominently displayed, and are given clear calls to action for communication.

- **Contextual Adaptations**: The design adapts to the specific context by prioritizing relevant details (product images, price, condition, shipping information) for the product and seller. The report option is included as a secondary option for moderation and security.


## Design System Compliance
- **Component Verification**: All componentNodeIds have been checked against the provided DESIGN_SYSTEM_DATA.
- **Variant Completeness**:  All components use appropriate variants from the design system.
- **Text Property Mapping**: Exact text layers from the design system schema have been used where possible.
- **Schema Adherence**: Properties are correctly mapped to component definitions.
