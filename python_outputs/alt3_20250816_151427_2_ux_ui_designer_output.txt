# DESIGN RATIONALE

## Component Intelligence Applied

Headline/Medium is used for section headers (Specifications, Description), and Body/Medium for the main content text.  Caption/Small is used for the "Last updated" timestamp. - **Spacing System**:  The `itemSpacing` of 16px in the main content container provides standard spacing between elements, creating visual breathing room. Padding of 16px on the sides and 20px on the top and bottom of the content container provides comfortable margins while adhering to the three-container mobile architecture pattern.

The Button (`10:3907`) component is used for both primary and secondary actions, with size and type variants reflecting their relative importance. - **Color Application**:  The primary color ("Primary/primary50") is used for the price to highlight its importance.  Neutral colors ("Neutral/neutral90" and "Neutral/neutral50") are used for body text and secondary text, respectively, for readability.

- **Layout Strategy**:  A vertical layout (`layoutMode: "VERTICAL"`) is used for the main screen to present information sequentially.  The three-container architecture (top navigation, content container, bottom navigation) ensures consistent and efficient use of screen space, conforming to mobile UI best practices.

## Domain Context Integration

- **Industry Patterns**: The design follows e-commerce patterns by clearly presenting product images, price, condition, specifications, description, and seller information. It also includes primary and secondary actions for contacting the seller and adding the product to a wishlist.

- **User Mental Models**:  The visual arrangement of the information mimics how users mentally model product listing details on e-commerce platforms, allowing for intuitive browsing and fast decision making. 

- **Contextual Adaptations**: The "Product Details" title in the app bar is a concise representation of the screen's purpose while the longer product name is placed at the top of the content area.

## Design System Compliance
