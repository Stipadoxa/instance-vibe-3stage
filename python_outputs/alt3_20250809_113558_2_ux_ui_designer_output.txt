**DESIGN RATIONALE:**

1. **Component Inventory Verification:** All component IDs have been verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout uses a vertical structure, prioritizing information hierarchy.  The hero section showcases images prominently, followed by the title and price.  Details are presented in a clear, concise list using the `list-item` component. The seller profile uses a horizontal layout for visual balance, with the avatar and information side-by-side. The description is in its own section, and the contact button is placed at the bottom for clear call to action.  Padding and spacing are consistent throughout, using the suggested patterns for full-width containers.

3. **User Flow Reasoning:** The user flow is designed to be intuitive. Key information (images, title, price) is presented upfront. Detailed specifications and seller information are presented below. The description provides further context, and finally, the "Contact Seller" button offers clear next steps.

4. **Component Choices:**  The `list-item` component was chosen to display product details due to its ability to handle both overline and headline text cleanly.  The `text area` component was selected for the description due to its support for multi-line text input. The `button` component's green variant provides clear visual emphasis on the contact seller action.

5. **Component Gaps:** The design system lacks a suitable carousel component for showcasing multiple product images.  This was addressed using a `layoutContainer` with a horizontal `layoutWrap: "WRAP"` mode and multiple instances of the `native-rectangle` component with image fills to simulate a carousel.  No star rating component exists; this was not implemented.

6. **Visual Hierarchy:** Visual hierarchy is established through typography (using design system text styles), color (using primary and secondary colors for emphasis), and layout positioning.  Prominent elements (images, title, price) are positioned at the top, while less critical details are placed below.

7. **Variant Selection Rationale:** Variants were selected to suit the information presented and for context: For the list items, the 2-line variant was used for the description and 1-line for other specifications.  For the button, the large filled green variant ensures high visibility and a clear call to action.


8. **Native Element Justification:** `native-rectangle` elements were used for image placeholders within the image carousel due to the absence of an image carousel component.
