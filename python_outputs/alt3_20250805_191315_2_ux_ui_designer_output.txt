**DESIGN RATIONALE:**

1. **Component Inventory Verification:**  All component IDs used below have been verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout uses a vertical structure. The top section features a full-width image gallery, providing immediate visual appeal. Below, key information (title, price, condition) is displayed prominently using a horizontal layout. Seller information (including rating, implemented as individual star icons) and product description follow in vertical sections, creating a clear visual hierarchy.  Consistent spacing (16px padding and 8px spacing where appropriate) is used throughout for readability. Full-width elements use `"horizontalSizing": "FILL"` to ensure they extend across the entire screen width.

3. **User Flow Reasoning:** The user flow is designed to be intuitive, starting with visually engaging product images.  Important details are presented first, followed by secondary information. The "Contact Seller" button provides a clear call to action.

4. **Component Choices:** 
    * **Image Gallery:** I used `native-rectangle` with image fills for image placeholders because there's no dedicated "image gallery" component. This strategy allows for flexible image sizing and a full-width presentation.  While `10:7816` (IMAGES) exists, it does not support the required gallery functionality.
    * **Button:** The `10:3907` (Button) component is ideal for the "Contact Seller" button. The `Size:Medium`, `Type:Filled`, `Color:Green` variants provide a clear and prominent action.
    * **Rating:** No dedicated rating component (`native-rating` is forbidden). The rating is implemented using multiple `635:4752` (icon/star) icons with `Star:full`/`Star:half` variants to achieve the star rating functionality.


5. **Component Gaps:** The design system lacks components for an image gallery and a star rating system.  I created a workaround for the image gallery using `native-rectangle` components with image fills and for the star rating using multiple instances of the `635:4752` star icon. This allows a visually appealing presentation while adhering to design system restrictions.

6. **Visual Hierarchy:** Visual hierarchy is established through a combination of layout and typography.  Large, prominent title and price text are placed above smaller, less important details to guide the user's attention.  The images are placed first to provide a strong visual hook.

7. **Variant Selection Rationale:** I selected `Size:Medium` for the "Contact Seller" button (`10:3907`) because it offers a balanced size for mobile screens.  For the star rating icons (`635:4752`), I used `Star:full` and `Star:half` variants depending on the rating level to represent a full or half-filled star accurately.

8. **Native Element Justification:**  The use of `native-rectangle` for the image gallery was necessary to provide the requested full-width image display and the ability to show multiple images. The system did not offer a component capable of this layout.  Similarly, the use of multiple `635:4752` (icon/star) components was necessary to meet the user's requirement for a visual rating system.
