# DESIGN RATIONALE

## Component Intelligence Applied
- **Typography Hierarchy**:  Headline/Large is used for the main product title to ensure prominence. Headline/Medium is used for section headers (Specifications), and Body/Medium and Body/Small for the body text and smaller details respectively, creating a clear visual hierarchy.
- **Spacing System**:  Consistent spacing tokens (8, 12, 16) are used throughout, creating visual harmony and separating content blocks effectively.  The root container has no spacing to allow for edge-to-edge navigation bars. Content padding is within the content container.
- **Component Behavior**: The app bar uses the "appbar" component with the back arrow visible and other trailing icons hidden.  The image carousel leverages the "IMAGES" component, allowing for a smooth image-swiping experience. Buttons utilize the "Button" component, with the "Contact Seller" button styled as a primary action. The wishlist icon is used, and a fallback is implemented for the report icon which uses a context menu (not shown as its structure is extensive and would exceed the output requirements).
- **Color Application**:  The color palette is drawn entirely from the provided design system, ensuring brand consistency. Primary colors are used for the price and CTA buttons, neutral colors for text, and secondary colors for less critical elements.
- **Layout Strategy**: The three-container mobile architecture is strictly followed. The screen uses a vertical layout for its main structure.  A horizontal layout is used within sections, particularly in the specifications.

## Domain Context Integration
- **Industry Patterns**: The design incorporates common e-commerce patterns, such as product image carousels, pricing displays, seller ratings, detailed specifications, and clear CTAs.
- **User Mental Models**:  The layout is intuitively organized, placing key information (price, condition, seller details) at the top of the screen, followed by detailed specifications and CTAs. The image carousel is immediately visible to showcase the product.
- **Contextual Adaptations**:  The app bar is adapted to show a short navigation-context title and the full product title is displayed in the main content.


## Design System Compliance
- **Component Verification**: All componentNodeIds are present in DESIGN_SYSTEM_DATA.
- **Variant Completeness**: All used components have variants completely specified with valid values.
- **Text Property Mapping**: Exact textLayers names (where appropriate) from the design system schema are used.
- **Schema Adherence**: Properties match component definitions exactly.
