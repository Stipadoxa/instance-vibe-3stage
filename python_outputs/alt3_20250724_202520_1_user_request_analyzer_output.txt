Domain & Context
Domain: Business/SaaS domain, specifically user account management within a mobile application.  Context is Cars45 auction platform.
User Type: Registered users of the Cars45 mobile app managing their account settings.  This includes both personal and business users.
Platform Impact: Mobile platform necessitates concise information display, clear touch targets, and a streamlined navigation system.  Limited screen real estate requires efficient use of space.

User Requirements
Primary Goals: Access and modify personal and business account information, manage notification preferences, and securely log out.
Secondary Goals:  Understand the meaning of "Verified ID" badge, easily change account settings (language, contact information, password), and securely delete their account.
User Constraints: Limited screen real estate on mobile device, reliance on touch interaction, potential for interruptions, and need for clear visual hierarchy and guidance.
Ready for handoff if: All settings options from the reference image are present and functional, top and bottom navigation bars are implemented according to design system, and all interactive elements have appropriately sized touch targets.

Content Requirements
Critical:  Personal details section, Business details section, Access to Cars45 auction toggle, "Verified ID" badge explanation, Change phone number option, Change email option, Change language option, Disable chats toggle, Disable Feedback toggle, Manage notifications option, Change password option, Delete my account permanently option, Log out option.
Supporting:  "What is it?" explanation for "Verified ID" badge (potentially a modal or help screen).
Contextual:  Active/Inactive status for the "Cars45 Auction" access setting.

Structural & Component Directives
* Layout: Vertical list of settings options, following the pattern in the reference image.  Top and bottom app navigation bars as specified.
* Components:  Use existing app components for list items, toggles, and buttons.  List items should follow the style of the reference image, possibly with an arrow icon to indicate expandable sections.  The "Verified ID" badge status could be represented visually.
* Styling:  Adhere to the existing app's color scheme, fonts, and spacing.  Use consistent styling for toggles, buttons, and text elements.

Screen Requirements
Primary Actions:  Navigate to different setting sections, toggle settings on/off, save changes to settings.
Secondary Actions:  Access detailed explanations ("What is it?"), navigate to other app sections via top and bottom navigation.
Screen States: Default state (all settings displayed), loading state (if applicable), success state (after saving changes), error state (if applicable).
Feedback Needs: Clear visual feedback indicating changes saved, loading indicators, and error messages if necessary.
