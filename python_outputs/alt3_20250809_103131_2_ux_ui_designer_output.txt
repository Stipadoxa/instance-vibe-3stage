**DESIGN RATIONALE:**

1. **Component Inventory Verification:** All component IDs used in this design have been manually verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout prioritizes a vertical flow, starting with a hero image section showcasing multiple product photos.  Below, key information (title, price, condition) is presented prominently.  Seller information (profile, rating) follows, maintaining a clear visual separation. Finally, the description and action buttons complete the page.  Consistent 16px padding is used in content sections for visual breathing room.

3. **User Flow Reasoning:** The user is presented with a visually appealing overview of the product. Key details are easy to scan before committing to reading the full description.  The contact button encourages interaction and purchase.

4. **Component Choices:** I chose the `product card` component (ID: 147:3743) for its suitability in displaying product information concisely.  It offers efficient display of images, title, price, and attributes.

5. **Component Gaps:** The design system lacks a dedicated component for displaying a seller's rating.  I created a workaround using multiple `icon/star` components (ID: 635:4752) to represent the star rating visually.

6. **Visual Hierarchy:** Visual hierarchy is established through the use of typography (headline, title, label styles from DESIGN_SYSTEM_DATA), spacing, and color.  Prominent elements (title, price, hero images) are given more visual weight through size and positioning.

7. **Variant Selection Rationale:** For the `Button` component (ID: 10:3907), the "Filled" type with "Large" size and "Green" color was chosen to emphasize its primary action role ("Contact Seller"). The "Default" state ensures a clear visual indication of its functionality.

8. **Native Element Justification:**  `native-rectangle` components with image fills are used to display multiple product images.  This is necessary because there isn't a designated component for image galleries within the design system.
