Domain & Context
Domain: Business/SaaS domain with user account management.  Specific context is a mobile application for Cars45 auction.
User Type: Users of the Cars45 mobile app needing to manage their account settings and preferences.  This includes both personal and business users.
Platform Impact: Mobile users need a concise and easily navigable settings page with clear touch targets.  Screen real estate is limited, so efficient grouping and presentation of settings is crucial.


User Requirements
Primary Goals: Access and modify account settings (personal and business details, communication preferences, account security), manage auction access.
Secondary Goals: Understand the meaning of "Verified ID" badge, easily log out of the application.
User Constraints: Limited screen space on mobile devices, reliance on touch interactions, potential for interruptions during use.
Ready for handoff if: All settings from the reference image are included, grouped logically as shown; top and bottom navigation is implemented using existing app components;  touch targets meet minimum size requirements; text is legible and clear.


Content Requirements
Critical: Personal details (editable), Business details (editable), Access to Cars45 auction (toggle), "Verified ID" badge (explanation link), Change phone number (input field), Change email (input field), Change language (selection list), Disable chats (toggle), Disable Feedback (toggle), Manage notifications (link to notifications settings screen), Change password (link to password change screen), Delete my account permanently (confirmation prompt), Log out (action).
Supporting:  "What is it?" link for "Verified ID" badge.
Contextual:  Success/failure messages after saving changes.


Structural & Component Directives
* Layout:  List-based layout, grouped settings as per reference image, uses existing app components
* Components: Existing app navigation components for top and bottom navigation, list items for each setting with clear labels and actions (toggle, input field, link, etc.).
* Styling:  Uses existing app's color scheme and visual style consistent with the reference image.


Screen Requirements
Primary Actions:  Modify settings, access linked screens (notifications, password change), log out.
Secondary Actions: Access "What is it?" for "Verified ID" badge, Delete my account (after confirmation).
Screen States: Default state, loading state (if applicable, e.g., for saving changes), success/failure states after changes are made.
Feedback Needs:  Clear indication of successful changes saved, error messages for invalid input or failed actions, visual feedback for toggles (on/off).
