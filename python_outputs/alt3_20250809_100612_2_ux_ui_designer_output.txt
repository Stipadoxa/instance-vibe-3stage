**DESIGN RATIONALE:**

1. **Component Inventory Verification:** All component IDs used in this design have been manually verified against the provided `DESIGN_SYSTEM_DATA`.  No invented component IDs were used.

2. **Layout Decisions:** The layout uses a vertical structure for mobile-first design. The top section contains a carousel for product images, followed by sections for the title/price, condition, seller profile, description, and finally, a call to action button.  Spacing between sections is consistent at 16px, with 20px padding on the main content area for visual breathing room.

3. **User Flow Reasoning:** The user first sees a visual overview of the product with multiple images. Then, critical information (title, price, condition) is immediately presented. Seller details and ratings provide trust and context. The detailed description allows for informed decision-making. The prominent "Contact Seller" button completes the user flow, encouraging engagement.

4. **Component Choices:**  The `product card` component (ID: `147:3743`) was chosen to handle much of the product information display, improving consistency. This is preferred over a series of native elements, which would be more time-consuming and less consistent with the overall design system. The `Button` component (ID: `10:3907`) will provide a user interface for the contact seller function.

5. **Component Gaps:** The design system lacks a dedicated image carousel/gallery component and a star rating component.  For the image gallery, I've used a `layoutContainer` with horizontal auto-layout and image fills on native rectangles to simulate a carousel (images are manually arranged in the specification below for clarity, a dedicated carousel implementation would require additional steps). For the star rating, I've implemented it using multiple `native-circle` elements to represent stars.

6. **Visual Hierarchy:** Visual hierarchy is established through component styling, section arrangement, and typography. The product title and price are prominently displayed using larger text sizes from the design system's text styles.  Seller information and product description have a smaller font size, but still maintain readability. Section dividers clearly separate information.

7. **Variant Selection Rationale:** For the "Contact Seller" button (`10:3907`), I chose the `Size: "Medium"`, `Type: "Filled"`, `State: "Default"`, and `Color: "Green"` variants.  This balances visual prominence with design system consistency. Other variants were excluded based on their current usability in this context.

8. **Native Element Justification:** Native rectangles with image fills were necessary to implement the image gallery functionality due to lack of a dedicated component.  Native circles are used for the star rating to mimic visual representation of a star rating component.
