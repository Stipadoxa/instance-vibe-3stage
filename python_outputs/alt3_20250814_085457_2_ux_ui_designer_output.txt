## SECTION 1: DESIGN RATIONALE

# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish clear visual hierarchy.  "Headline/Large" is used for the main product title to provide prominence. "Title/Medium" styles section headers (like "Seller Profile" and "Specifications"), while "Body/Large" is used for the main product description and "Body/Medium" for supporting text within the specifications.  "Caption/Small" is used for the "Last Updated" timestamp.  This ensures readability and guides the user's attention effectively.

- **Spacing System**: The `itemSpacing` and padding values are carefully chosen to create visual harmony and separation between content blocks.  `itemSpacing: 16` is used between major sections (e.g., image carousel, price, description), providing a comfortable breathing space.  Internal padding within components is respected and adjusted as necessary to maintain consistency. Section containers use `paddingTop/Bottom: 20` and `paddingLeft/Right: 16` for standard spacing.

- **Component Behavior**: The `appbar` component is used for the top navigation, with the back arrow icon (`635:4717` with `style: back`)  as a leading icon.  The `Bottom navigation` component (`10:4724`) is used for the bottom navigation bar, displaying icons for Home (`635:4317`), Search (`635:4107`), Messages (`635:4364`), and Profile (`635:4162`).  The `Button` component (`10:3907`) is used for the primary "Contact Seller" action (with `Size: Large`, `Type: Filled`, `Color: Green`) and the secondary "Report Listing" action (`Size: Medium`, `Type: Outlined`, `Color: Error/error50`). The lack of a suitable carousel component necessitates the use of a native image container with a horizontal scrolling mechanism.

- **Color Application**: The color palette is carefully chosen to align with the design system and e-commerce best practices.  The primary brand color (`Primary/primary50: #00b53f`) is used sparingly for key interactive elements such as the "Contact Seller" button, bottom navigation bar, and price.  Neutral colors (`Neutral/neutral90: #304049` and `Neutral/neutral50: #6c8ea0`) are used extensively for text and background elements.  This approach ensures good contrast and visual clarity.

- **Layout Strategy**:  A vertical layout is used to organize information logically. Sections are stacked vertically, with appropriate spacing.  The image carousel utilizes a horizontal layout within its container. The `primaryAxisSizingMode: AUTO` and `counterAxisSizingMode: FIXED` are used where appropriate for content hugging heights and full-width components.


## Domain Context Integration

- **Industry Patterns**: The design adheres to common e-commerce patterns for product detail pages, including a prominent hero image, clear pricing, detailed descriptions, and seller information to build trust.

- **User Mental Models**: The layout flows naturally from top to bottom, mirroring typical reading patterns. The most important information (title, price, images) is presented upfront.  Specifications are collapsible to reduce clutter on mobile screens.

- **Contextual Adaptations**: The design is mobile-first, ensuring all interactive elements are large enough for comfortable touch interactions.  The specifications table is collapsible to optimize screen real estate.


## Design System Compliance

- **Component Verification**:  All componentNodeIds have been checked against the `DESIGN_SYSTEM_DATA` and confirmed to exist.  Fallback strategies have been implemented for missing components, specifically for the image carousel.

- **Variant Completeness**: All necessary variants for the chosen components are included, with values validated against `DESIGN_SYSTEM_DATA`.

- **Text Property Mapping**:  The UI specification uses the exact `textLayers` names from the design system schema.

- **Schema Adherence**: All component properties adhere strictly to their respective definitions within the schema.
