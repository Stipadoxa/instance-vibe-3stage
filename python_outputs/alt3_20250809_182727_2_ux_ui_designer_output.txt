**DESIGN RATIONALE:**

1. **Component Inventory Verification:** All component IDs referenced below have been manually verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout uses a vertical structure, prioritizing information hierarchy. The hero image section takes up significant vertical space, followed by product details, seller information, and a call to action.  Spacing is consistent (16px padding, 8px between sections) throughout using `layoutContainer` patterns.

3. **User Flow Reasoning:**  The user first sees the hero image and product title, quickly grasping the item. Next, price, condition, and key specifications are presented to build interest. The seller's profile and rating build trust. A full description satisfies informational needs, ending with a prominent "Contact Seller" button.

4. **Component Choices:** I chose the `product card` component (147:3743) for its versatile structure and ability to display multiple images and various attributes. This was preferable to a simpler card component because it allowed for flexible presentation of the product information.

5. **Component Gaps:** The design system lacks a dedicated seller rating component.  I used multiple `star` icons (635:4752) to create a visual rating representation, which is a common and effective workaround.

6. **Visual Hierarchy:** Visual hierarchy is established through size, position, and color contrast.  The product hero image has the highest prominence, followed by the title and price in larger text sizes and primary colors.

7. **Variant Selection Rationale:** For the "Contact Seller" button (10:3907), I selected the `Size: Medium`, `Type: Filled`, `Color: Green`, and `State: Default` variants to provide a clear and consistent call to action.

8. **Native Element Justification:**  I used `native-text` for displaying the product description and additional metadata because the design system text styles don't offer sufficient flexibility in styling for paragraph text. The image carousel (multiple `native-rectangle` components with image fills) was chosen because a dedicated image carousel component wasn't present in the system.
