# DESIGN RATIONALE

## Component Intelligence Applied
- **Typography Hierarchy**:  Headline/Large is used for the main product title, Headline/Medium for price and condition, Title/Medium for seller information, Body/Medium for descriptions and specifications, and Caption/Small for the update timestamp. This hierarchy ensures clear visual distinction between different levels of information.

- **Spacing System**:  Consistent spacing tokens (8, 12, 16) are used throughout to maintain visual harmony. ItemSpacing of 16 is used between major sections, while 8 and 12 are used for closer related elements. Padding is applied within the main content container to provide breathing room around the content.

- **Component Behavior**:  The image carousel uses a customizable component to allow for multiple images, and zoomability is accounted for in the design. The rating system leverages a dedicated rating component. Variants are selected based on context (e.g., large filled button for the primary action).

- **Color Application**:  Primary colors are used for price and primary actions.  Neutral colors (Neutral/neutral90 and Neutral/neutral80) are utilized for main text, maintaining readability.


- **Layout Strategy**: The three-container mobile architecture (top navigation, content, bottom navigation) is strictly adhered to, with zero padding on the root container and layoutGrow: 1 applied to the main content container.  A vertical layout is used for the main content area, aligning to user's reading direction, while horizontal layouts are used for grouping related elements.

## Domain Context Integration
- **Industry Patterns**: The design follows standard e-commerce patterns for product detail pages, including a prominent product image, clear pricing, seller information, specifications, and customer reviews.

- **User Mental Models**: The layout is designed to intuitively match user expectations in an e-commerce setting, guiding the user smoothly through the information flow from product image to specification to review to contact.

- **Contextual Adaptations**:  The app bar title is concise ("Product Details"),  following the recommended character limit to prioritize navigation context over title length.

## Design System Compliance
- **Component Verification**: All componentNodeIds are validated against the provided DESIGN_SYSTEM_DATA.
- **Variant Completeness**: All components use appropriate variants and values as described in the DESIGN_SYSTEM_DATA.
- **Text Property Mapping**: Exact textLayers names were used for components where possible.
- **Schema Adherence**: All properties and values adhere strictly to the provided design system schema.
