**DESIGN RATIONALE:**

1. **Component Inventory Verification:** I have verified all component references against the provided design system data.

2. **Section Planning:** The login page follows the "Login" section pattern: logo → form → social → links → security → footer.  The logo establishes brand identity. The form handles email/password input and primary login.  Social login provides alternative authentication, while links direct to supplemental information. Security and footer elements are omitted for the MVP.

3. **Layout Decisions:** A vertical layout is used for a clear top-to-bottom flow. The logo is at the top for immediate brand recognition. The form is centrally positioned for focus. Social login options are placed below the form for easy access. Links to the Terms of Service and Privacy Policy are placed at the very bottom. Content padding of 16px is added to Level 1 layout containers.

4. **User Flow Reasoning:** The user is first presented with branding and instructions. They proceed to fill out the login form, click "Sign In," and then will be taken to the app.  If they experience issues, they have the option to reset their password or create an account.

5. **Component Choices:** I chose the `text input` component for email and password fields over `native-text` because it provides built-in input validation and a better user experience. I chose the `Button` component for the "Sign In" button because it adheres to design standards.

6. **Component Gaps:**  The design system lacks a dedicated social login component.  I created a workaround using `native-rectangle` elements to represent social icons, allowing for visual cues while adhering to the specified native element limitations.  This provides a minimal functional solution.

7. **Visual Hierarchy:** The `Title/Large` text style for the welcome message establishes prominence.  The form elements are the next most important, presented with clear labeling.  Social login and links maintain a lower visual weight using smaller text styles.

8. **Variant Selection Rationale:** I chose the "Filled" variant for the input fields because this is usually preferred for login forms and is standard for the design system.

9. **Native Element Justification:** The user request includes social login buttons, for which there is no component available. Native `native-rectangle` components are used as placeholders for social media logos for quick visual feedback to the user to provide a functional solution within the API constraints. This allows the user to quickly identify the social login option, even if it is not fully-featured.
