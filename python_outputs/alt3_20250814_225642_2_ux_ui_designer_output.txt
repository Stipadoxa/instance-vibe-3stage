## SECTION 1: DESIGN RATIONALE

# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish a clear visual hierarchy.  "Headline/Large" is used for the main product title, providing prominence and clear identification of the item. Section headers utilize "Title/Medium" for a strong but less dominant emphasis. Body text, including the product description and specifications, employs "Body/Medium" for readability.  "Caption/Small" is reserved for less important information like the listing age. This consistent application of text styles ensures that the information is easily scanned and understood.


- **Spacing System**: The design utilizes a consistent spacing system based on the provided `itemSpacing` values.  `itemSpacing: 16` is used between major sections of content, promoting visual separation and improved readability.  Within sections, `itemSpacing: 8` is used for tighter grouping of related elements. Padding within containers is applied consistently based on the section hierarchy (see Layout Strategy).  This ensures a clean and organized layout while maintaining optimal spacing relationships.

- **Component Behavior**: The design system's `Button` component is used for both the "Contact Seller" (Primary/primary50, Size: Large) and "Add to Favorites" (Secondary/secondary50, Size: Medium) actions, ensuring consistency with the overall app style.  The "back" navigation icon is used as a left aligned icon in the `appbar` component. Icons for the bottom navigation are chosen semantically: "home", "search", "chat", "person" based on their typical meaning and usage in similar apps.  The `visibilityOverrides` property is not used in this design as no elements need to be conditionally hidden. Icon swaps are also not required.

- **Color Application**:  The color palette adheres strictly to the provided design system.  "Primary/primary50" (#00b53f) is the main accent color, utilized in the primary call-to-action ("Contact Seller"). Neutral colors ("Neutral/neutral90" for primary text and "Neutral/neutral50" for secondary text) ensure readability against the "White/white100" background.  This consistent color usage reinforces branding and enhances the overall visual appeal.

- **Layout Strategy**: The screen follows the critical three-container mobile app architecture. The top navigation bar (`appbar` component with a back button) sits at the top, edge to edge. The main content area uses a vertical layout (`layoutContainer` with `layoutMode: VERTICAL`, `itemSpacing: 16`, and appropriate padding) to present information logically.  The bottom navigation bar (`Bottom navigation` component) remains fixed at the bottom, allowing for quick navigation throughout the app.  `layoutGrow: 1` on the content container ensures the bottom nav stays at the bottom regardless of content length. This structure prioritizes usability and maintain consistent visual patterns.


## Domain Context Integration

- **Industry Patterns**: The design incorporates common e-commerce patterns like product image carousels, detailed product descriptions, pricing, seller profiles with ratings and reviews, and clear call-to-action buttons. These familiar elements help users quickly understand the purpose and functionality of the screen, reducing cognitive load.

- **User Mental Models**: The visual layout follows a typical e-commerce product detail page layout. Information is organized logically from top to bottom, starting with prominent product information (images, title, price) and then moving to secondary details (seller information, product description, specs, shipping). This aligns with user expectations based on their previous experiences with online marketplaces.

- **Contextual Adaptations**: The image carousel is a significant element and designed for optimal viewing on mobile screens. The product description is formatted for mobile reading.


## Design System Compliance

- **Component Verification**: All componentNodeIds used in the JSON specification below are validated against the provided `DESIGN_SYSTEM_DATA`.

- **Variant Completeness**:  All specified variants are fully detailed and valid according to the design system data.

- **Text Property Mapping**:  Text content is mapped to relevant `textLayers` within components, or where necessary, `textStyle` properties of `native-text` elements are provided using correct design system style names.

- **Schema Adherence**: All properties in the JSON adhere exactly to the component definitions within `DESIGN_SYSTEM_DATA`.
