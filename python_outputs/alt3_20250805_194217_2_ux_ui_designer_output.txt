**DESIGN RATIONALE:**

1. **Component Inventory Verification:**  All component IDs used in the design specification below have been verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout uses a vertical structure for a clear top-to-bottom flow of information. The image gallery occupies full width at the top for visual impact, followed by product details and seller information in separate sections with appropriate padding for readability.  Horizontal sizing is used to fill the container width for full-width elements like the image gallery and some text elements.

3. **User Flow Reasoning:** The user first sees the product images, followed by the title and price, then important details like condition and seller rating. This hierarchy directs the user to make a quick assessment before reading the full description and deciding to contact the seller.

4. **Component Choices:**
    - **`10:3907` (Button):** Chosen for "Contact Seller" button due to its prominent design and ease of use. Alternatives considered include using native elements for a button, but the component offers built-in accessibility and styling.
    - **`10:7773` (Avatar):** Selected to display seller's profile image, offering consistent sizing and styling.  Alternatives include `10:7798` (avatar/single), but `10:7773` is visually more prominent.  
    - **`10:7816` (IMAGES):** For the image gallery, as it is meant to display multiple images. There is no apparent alternative.
    - **`10:8619` (text area):** Used for the product description field due to the need for multi-line text input. No other component perfectly fits this requirement.


5. **Component Gaps:**  The design system lacks a dedicated star rating component (`native-rating` is forbidden).  I implemented a workaround using five `635:4130` (icon/star_rate) components with variants to represent a star rating. This approach maintains a consistent visual style while addressing the functionality.


6. **Visual Hierarchy:**  Visual hierarchy is established through size, placement, and color. The product images at the top hold most prominence. The title and price are larger than the description, and the “Contact Seller” button is visually distinct.

7. **Variant Selection Rationale:**
    - **Button (`10:3907`):** Variants `"Size": "Medium", "Type": "Filled", "State": "Default", "Color": "Green"` were selected for optimal visibility and clear action indication.
    - **Avatar (`10:7773`):** `"Photo": "Yes"` is used to ensure an image is displayed for the seller's profile.

8. **Native Element Justification:**  Native elements were necessary to achieve the layout structure. I used `native-rectangle` and `native-text` components to structure the screen content and to accommodate the product title and price. `native-text` component has also been used to display product description.
