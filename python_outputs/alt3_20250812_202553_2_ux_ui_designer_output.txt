# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design utilizes a clear typography hierarchy based on the provided design system.  `Headline/Large` is used for the main product title, conveying its primary importance.  `Title/Medium` styles section headers (like "Seller Profile" and "Specifications"), while `Body/Large` and `Body/Medium` are used for the product description and supporting details respectively. `Caption/Small` is used for less important metadata like the posting date. This ensures readability and guides the user's visual attention effectively.

- **Spacing System**:  The spacing system is designed to create visual harmony and improve readability.  `itemSpacing` values of 16 are used between major sections to provide sufficient visual breathing room.  An `itemSpacing` of 8 is used within sections to group related content closely.  Padding values of 16 are consistently used within content containers for comfortable reading.

- **Component Behavior**:  The design leverages the `Button` component for both primary ("Contact Seller") and secondary actions ("Ask Seller a Question", "Report Listing"). Variants are used appropriately: "Large" size for primary CTAs, and "Medium" size for secondary.  The image carousel is implemented using the `IMAGES` component, adapting to the number of images provided. The Seller Profile utilizes the `avatar` component with image placement, and the star rating is handled using a rating component (assuming it exists; a fallback will be implemented if not). The absence of a dedicated product card component necessitates the use of a `list-item` component with appropriate variants to represent product attributes.

- **Color Application**: The color palette adheres strictly to the design system.  `Primary/primary50` (#00b53f) is used sparingly for accentuation (price), while `Neutral/neutral90` (#304049) and `Neutral/neutral50` (#6c8ea0) form the core text hierarchy. Backgrounds use `White/white100` (#ffffff) for maximum readability and to align with common e-commerce patterns.

- **Layout Strategy**: The layout uses a vertical `layoutContainer` to organize information logically.  A `HORIZONTAL` container is used for the image carousel to present images efficiently.  The primary axis sizing mode is "AUTO" to dynamically adjust the height of sections, while the counter axis sizing mode is "FIXED" to maintain consistent width on the 375px viewport.

## Domain Context Integration

- **Industry Patterns**: The design mirrors established e-commerce product page conventions.  Key information (price, condition, images) is prominently displayed at the top.  Seller information, description, specifications, and actions follow a natural flow.

- **User Mental Models**: The design aims to meet user expectations by providing a clear and concise presentation of product details.  Prominent visual cues guide users toward essential information and actions, aligning with established mental models for product browsing on e-commerce marketplaces.

- **Contextual Adaptations**: The design adapts to the specific content provided by the user, dynamically adjusting image carousel size and utilizing the available components to represent the unique attributes of a used laptop listing.  Fallback mechanisms were employed for missing components.

- **Component Verification**: All componentNodeIds were verified against the design system data.  Where needed, fallback mechanisms were used.
- **Variant Completeness**: All relevant variants for the chosen components are completely specified.
