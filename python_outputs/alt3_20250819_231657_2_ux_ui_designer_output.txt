# DESIGN RATIONALE

## Component Intelligence Applied
- **Typography Hierarchy**:  Headline/Large is used for the product title to give it prominence. Headline/Medium is used for the price to make it stand out. Title/Medium is used for section headers (Specifications, Description) and seller's name. Body/Medium is used for the majority of the descriptive text, while Body/Small is used for the review count. Caption/Small is used for the "Last updated" timestamp.

- **Spacing System**:  The `itemSpacing` within containers is adjusted to create visual separation between content blocks while maintaining a clean and uncluttered layout.  Padding is used consistently within the main content area to ensure appropriate spacing around text and components.

- **Component Behavior**:  The image carousel is implemented using a zoomable image component. The star rating component dynamically displays the seller's rating.  Buttons use appropriate sizes (Large for primary, Medium for secondary) and styling based on their actions.

- **Color Application**: Primary colors are reserved for the price and primary action (Contact Seller) button to draw attention to them. Neutral colors are used for the majority of the text for improved readability.

- **Layout Strategy**:  The three-container mobile architecture (top nav, content, bottom nav) is strictly followed for optimal mobile app structure and UX.  Vertical layout is used for most containers to reflect natural content flow.


## Domain Context Integration
- **Industry Patterns**: The design mirrors standard e-commerce product detail page patterns, including prominent display of the product image, price, condition, seller information, specifications, description, and prominent call to action.

- **User Mental Models**: The layout prioritizes key information upfront (images, title, price, seller rating) to facilitate quick decision-making.  Specifications and description are organized in logical sections.

- **Contextual Adaptations**: Components from the design system are reused wherever possible to maintain consistency across the application and reduce development effort.


## Design System Compliance
- **Component Verification**: All componentNodeIds used are verified against the provided `DESIGN_SYSTEM_DATA`.
- **Variant Completeness**:  All components use variants appropriate to their context.  For instance, the app bar uses the "Default" type, and the button utilizes "Large" size and "Filled" type.
- **Text Property Mapping**:  Text content is mapped to appropriate text slots within components where available.  Where native text is required,  design system text styles are employed consistently to maintain typography.
- **Schema Adherence**: All components and properties are used according to their definitions within the provided `DESIGN_SYSTEM_DATA`.
