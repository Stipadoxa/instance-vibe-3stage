# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish a clear visual hierarchy.  "Headline/Large" is used for the main product title, providing prominence. "Title/Medium" styles section headers (e.g., Specifications, Seller Profile), and "Body/Medium" is used for descriptive text to ensure readability.  "Caption/Small" is used for the "Posted" date.

- **Spacing System**:  `itemSpacing` values of 16 are used between major sections (Gallery, Specifications, Seller Profile, Description) to create visual breathing room.  `itemSpacing` of 8 is used within the Specifications section to group related specifications.  Consistent padding (16px) is used within the main content container to provide a comfortable reading experience.

- **Component Behavior**: The `appbar` component is used for top navigation, with a "back" icon swap applied to the `leading-icon`. The `Bottom navigation` component is used for bottom navigation, displaying the appropriate icons.  A `Button` component (Type: "Filled", Size: "Large", Color: "Green") is used for the primary "Contact Seller" action. The "Report Listing" action is implemented as a small, discreet icon button in the upper right corner of the screen (using a native element approach as a fallback strategy due to the lack of a pre-built "small icon button" component).


- **Color Application**: The color palette is chosen for visual clarity and to reinforce the e-commerce context. "White/white100" is used as the main background color for optimal readability. "Neutral/neutral90" is used for primary text, while "Neutral/neutral50" is used for secondary text, such as specifications and descriptions. The primary action button uses "Primary/primary50" to maintain brand consistency.

- **Layout Strategy**: A three-container layout (top navigation bar, main content, bottom navigation bar) is used as per the defined mobile app architecture pattern. The main content area uses a vertical layout (`layoutMode: "VERTICAL"`) to organize information logically.  The gallery uses a horizontal layout for image presentation.


## Domain Context Integration

- **Industry Patterns**: The design incorporates typical e-commerce patterns, such as product image galleries, detailed specifications, pricing, seller ratings, and a clear call to action ("Contact Seller").

- **User Mental Models**: The layout is designed to be intuitive, following user expectations for e-commerce product detail pages. Key information is presented prominently, and actions are clearly visible.

- **Contextual Adaptations**:  The design adapts the generic components to fit the specific context of viewing a used laptop listing.  For example, the image gallery is adapted to display four relevant images.


## Design System Compliance

- **Component Verification**: All componentNodeIds used are verified to exist within the provided `DESIGN_SYSTEM_DATA`.
- **Variant Completeness**: All used components are instantiated with a complete set of appropriate variants.  Fallbacks are used where necessary.
- **Text Property Mapping**: Text properties in all components are mapped to the corresponding text layer names from the component schema.
- **Schema Adherence**: All component properties and variants adhere strictly to the specifications in the `DESIGN_SYSTEM_DATA`.
