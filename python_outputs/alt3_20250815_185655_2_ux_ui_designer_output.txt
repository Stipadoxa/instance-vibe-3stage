# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design uses Headline/Large for the main product title, Title/Large for the price, Title/Medium for section headers (Specifications), and Body/Medium for descriptions and specifications details. Caption/Small is used for the "Updated" timestamp. This hierarchy ensures clear visual importance and scannability.

- **Spacing System**:  The `itemSpacing` of 8 is used within horizontal containers to separate closely related elements (price and updated timestamp).  A `paddingTop` and `paddingBottom` of 16 in the main content container provides standard spacing around content blocks.  Sections are separated by an `itemSpacing` of 16 within the main vertical container.

- **Component Behavior**: The `10:7816` (IMAGES) component is used for the image carousel, allowing for zoomable images. The "Button" component (`10:3907`) is used for both primary (Contact Seller) and secondary (Add to Watchlist) actions, with appropriate size and type variants for visual distinction.  The appbar component (`10:5620`) is configured with a back arrow icon for navigation.

- **Color Application**: Primary/primary50 is used for the price to highlight its importance. Neutral/neutral90 is used for main headings and labels, Neutral/neutral50 for supporting text, and Primary/primary50 for CTAs.  This ensures consistent brand application while maintaining readability.

- **Layout Strategy**: A three-container vertical layout (top navigation, content, bottom navigation) is used to adhere to the specified mobile app architecture pattern, ensuring navigation elements are always visible and the content expands to fill the screen.


## Domain Context Integration

- **Industry Patterns**: The design incorporates common e-commerce patterns, such as product image carousels, pricing displays, seller profiles with ratings, detailed specifications sections, and clear call-to-action buttons.

- **User Mental Models**: The layout is intuitive, placing key information (price, condition, seller profile) prominently above the specifications and description, which is a familiar pattern in online marketplaces.  The visual hierarchy and clear separation of sections support users in quickly understanding the product details.

- **Contextual Adaptations**:  The app bar title is kept short ("Product Details") to comply with length restrictions, providing navigation context rather than showing the full product title. The full product title appears in a Headline/Large style within the main content for better readability.
