Domain & Context
Domain:  Business/SaaS domain with user account management; specifically, a mobile application for Cars45 auction.
User Type: Users of the Cars45 mobile app who need to manage their account settings and preferences. This includes both individual users and business users.
Platform Impact: Mobile users need a concise, easily navigable settings page with clear touch targets and minimal scrolling.  Limited screen real estate necessitates efficient grouping and organization of settings.

User Requirements
Primary Goals:  Access and modify personal information, manage account preferences, control notification settings, and securely logout.
Secondary Goals: Understand the meaning of "Verified ID" badge, easily access help or information related to settings options.
User Constraints: Small screen size, touch-only interaction, potential for distractions while using the app, need for intuitive navigation.
Ready for handoff if: All settings listed in the reference image are present and grouped logically;  navigation is intuitive and accessible via top and bottom app navigation;  all interactive elements meet minimum touch target size guidelines.

Content Requirements
Critical:  Personal details (editable), business details (editable), Access to Cars45 auction (toggle), "Verified ID" badge (with explanation), change phone number (editable), change email (editable), change language (selectable), disable chats (toggle), disable feedback (toggle), manage notifications (link to separate screen), change password (action), delete my account permanently (action), log out (action).
Supporting:  Clear visual grouping of settings (as per reference image).
Contextual:  "Active" status indicator where applicable,  "What is it?" help text for "Verified ID" badge.

Structural & Component Directives
* Layout:  Vertical list of settings grouped into logical sections with headers (as per reference image).
* Components: Use existing app components for list items, toggles, and other interactive elements.  Navigation should consist of standard top and bottom app navigation bars.
* Styling:  Maintain consistent styling with the existing app.  Use color scheme and visual hierarchy from the reference image.

Screen Requirements
Primary Actions:  Modifying setting values, initiating account deletion, logging out.
Secondary Actions: Accessing help/information for settings ("What is it?").
Screen States:  Default (all settings displayed), loading state (if applicable during settings modification).
Feedback Needs:  Confirmation of settings changes (visual cues),  loading indicators during processing (if applicable).
