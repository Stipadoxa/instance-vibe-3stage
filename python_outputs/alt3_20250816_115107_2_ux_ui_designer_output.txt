# DESIGN RATIONALE

## Component Intelligence Applied
- **Typography Hierarchy**:  Headline/Large is used for the main product title to provide prominence. Title/Large is used for the price to highlight its importance. Title/Medium is used for the seller's name and section headers (Condition, Specifications), providing a clear visual distinction. Body/Medium is used for the product description and specifications details for readability. Caption/Small is used for the "Last updated" timestamp to indicate its secondary importance.

- **Spacing System**:  The root container uses `itemSpacing: 0` to ensure the app bar and bottom navigation bar are flush with the edges.  The content container uses `paddingTop`, `paddingBottom`, `paddingLeft`, and `paddingRight` values of 16px to provide standard spacing around the content.  `itemSpacing: 16` is used within the vertical layout of the content section to provide sufficient visual breathing room between elements, and `itemSpacing: 8` is used between the seller's profile picture and name/rating for visual clarity.  `itemSpacing: 4` is used within the specification list to create visual grouping.

- **Component Behavior**: The `appbar` component's `visibilityOverrides` and `iconSwaps` properties are used to show the back arrow and hide other trailing icons, aligning with standard navigation patterns.  The image carousel component (`10:7816`) is used to display multiple product images effectively, taking advantage of its built-in image handling capabilities.  The `Button` component is used for both primary and secondary actions ("Contact Seller" and "View Seller Profile"), utilizing different variants (Filled vs Outlined) to signal their respective importance.

- **Color Application**: The primary brand color (`Primary/primary50`) is used for the price to grab attention.  Neutral colors (`Neutral/neutral90` and `Neutral/neutral50`) are used for the majority of the text, ensuring high readability.  The color palette selection aims for high contrast and clarity.

- **Layout Strategy**: A three-container architecture is employed (AppBar, Content, Bottom Navigation) for consistent mobile app layout, following the described pattern. Vertical layout is used for the main content section to present the information in a logical, top-to-bottom sequence. Horizontal layout is used for the seller profile to bring the profile picture and information closer together.


## Domain Context Integration
- **Industry Patterns**: The design follows standard e-commerce patterns, including a prominent product image, clear pricing, seller information, detailed product description, and clear call-to-action buttons.

- **User Mental Models**: The layout is designed to match user expectations for a product details page.  Important information is displayed prominently, and secondary information is organized logically and intuitively.

- **Contextual Adaptations**: The design leverages components suited to the e-commerce domain (image carousel, rating, button), ensuring a tailored user experience. The app bar title is concise ("Product Details"), focusing on navigation context rather than content-specific details, as per the guidelines.
