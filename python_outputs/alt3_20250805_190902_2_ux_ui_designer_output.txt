**DESIGN RATIONALE:**

1. **Component Inventory Verification:**  All component IDs used in the design specification below have been verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout uses a vertical arrangement for a clear top-to-bottom flow of information.  The image gallery section utilizes a horizontal `layoutContainer` with `layoutWrap: "WRAP"` to allow images to wrap across multiple lines based on screen size.  Other sections (title, price, description, etc.) are vertically stacked with appropriate spacing for readability.  Full-width elements like the hero image utilize `"horizontalSizing": "FILL"` to ensure they span the entire screen width.  Fixed widths are used for specific elements to maintain consistency and prevent excessive stretching.

3. **User Flow Reasoning:** The user flow prioritizes quick information scanning and easy navigation to contact the seller. Key information (title, price, condition) is prominently displayed near the top. The image gallery encourages thorough product inspection. The description provides detailed information, and the contact button provides clear means of engagement.

4. **Component Choices:**
    * **Image Gallery:**  The design system lacks a dedicated image gallery component. I opted to create this functionality using the `native-rectangle` component with image fills, arranged in a horizontally wrapping container. This was preferred over using individual image components due to the requirement for a flexible, multi-image display that adapts to different screen sizes.
    * **Contact Seller Button:** The `10:3907` ("Button") component is selected. It is preferred over creating custom buttons with native elements to maintain design system consistency. Variants "Type": "Filled", "Size": "Medium", "Color": "Green", "State": "Default" are chosen for a standard look and feel. 
    * **Seller Profile:** The design system does not provide an adequate seller profile component.  Instead, I'll use a combination of `native-circle` (for the avatar), `native-text` (for name and rating), and a custom implementation of star ratings using multiple `635:4752` ("icon/star") components.
    * **Divider:** The `10:8459` ("Divider") component is used with variant "Position": "horizontal", and "Type": "full-width" to separate sections for better readability and visual hierarchy.

5. **Component Gaps:** As mentioned above, a dedicated image gallery and a unified seller profile component are missing from the design system. The image gallery was built with native rectangles and auto layout, and the seller profile was implemented using combinations of native elements and existing components. These provide flexible layouts necessary for the product listing.

6. **Visual Hierarchy:** Visual hierarchy is established through prominent text sizes (title using the "Title/Large" text style and price using the "Title/Medium" text style), strategic use of spacing between sections, and color contrast to highlight key information.  The image gallery takes visual precedence.

7. **Variant Selection Rationale:**
    * **Button (10:3907):** The "Filled" type ensures a clear, prominent call-to-action. "Medium" size strikes a balance between readability and screen space. "Green" color connotes positive action. "Default" state is initially displayed.

8. **Native Element Justification:**  Native elements (`native-rectangle`, `native-circle`, `native-text`) are used to create the image gallery and elements of the seller profile because the design system doesn't offer dedicated components for these complex, adaptive layout needs. Native elements provide flexibility to precisely control the layout behavior that's needed for this UX.
