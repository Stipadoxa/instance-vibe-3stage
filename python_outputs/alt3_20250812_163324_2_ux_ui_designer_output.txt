# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design uses a clear typography hierarchy to guide the user's eye and emphasize important information.  "Headline/Large" is used for the main product title, providing visual prominence. "Title/Medium" styles section headers (Specifications, Description), creating a clear distinction from the main title and body text. "Body/Medium" is used for the majority of the descriptive text, ensuring readability.  "Label/Medium" is used for labels within the specifications table, maintaining consistency. "Caption/Small" is used for less important information like the listing date.


- **Spacing System**:  Consistent spacing is maintained using the design system's spacing tokens.  `itemSpacing: 16` is used between major sections (e.g., product title and image carousel, image carousel and specifications), providing visual breathing room. `itemSpacing: 8` is used within the specifications table for tighter grouping of rows.  Padding values are also applied consistently, with `paddingTop/Bottom: 20` and `paddingLeft/Right: 16` used for most content containers to ensure a comfortable reading experience.

- **Component Behavior**: The `list-item` component is used for the specifications table, leveraging its "2-line" variant to accommodate both attribute and value. The "Button" component is utilized for primary actions ("Contact Seller," "Add to Cart"),  employing the "Large" size for prominence. Secondary actions ("Report Listing," "View Seller Profile," "Save Listing") are implemented as "Button" components with the "Medium" size.  The avatar is implemented using the `avatar / big` component (fallback to native elements if unavailable) to display the seller's profile picture.

- **Color Application**: The design uses a consistent color palette based on the design system.  Primary colors (#00b53f) are used for calls to action (buttons), reinforcing their importance. Neutral colors are used for backgrounds and text (primarily #304049 and #ffffff) to ensure high contrast and readability. Secondary colors are used for less important elements.

- **Layout Strategy**: A vertical layout is used, following the typical mobile-first approach of organizing information top to bottom.  Containers are used to group related elements and manage spacing effectively.  The horizontal layout within the image carousel and specifications table is implemented using the appropriate `layoutContainer` patterns within the vertical layout.

## Domain Context Integration

- **Industry Patterns**: The design adheres to standard e-commerce product listing page conventions, ensuring a familiar and intuitive user experience. Key information (product title, price, images) is prominently displayed. Supporting information (specifications, description, seller details) is logically organized below.

- **User Mental Models**: The layout aligns with user expectations for product listing pages.  Users readily understand the flow of information (title, images, specifications, description, actions), reducing cognitive load and improving usability.

- **Contextual Adaptations**: No specific adaptations are needed as the provided components accommodate the given requirements.


## Design System Compliance

- **Component Verification**: All componentNodeIds have been verified against DESIGN_SYSTEM_DATA.  Fallback mechanisms have been employed where necessary.

- **Variant Completeness**: All necessary variants for the selected components are specified, with values selected to meet both user requirements and design system recommendations.

- **Text Property Mapping**: Exact textLayers names from the design system are used consistently for all text elements within the components.

- **Schema Adherence**: All component properties adhere strictly to the design system schema, ensuring design system compliance.
