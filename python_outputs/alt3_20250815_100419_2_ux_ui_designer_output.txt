# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to ensure clear visual hierarchy.  "Headline/Large" is used for the main product title, providing prominence and readability. "Title/Medium" styles section headers (like "Seller Profile" and "Specifications"), creating clear visual separation between information blocks. "Body/Large" is used for the main product description to maintain readability. "Body/Medium" is used for specifications to improve scannability and "Caption/Small" for the "Last Updated" timestamp.


- **Spacing System**: The `itemSpacing` and padding values are meticulously chosen to create visual harmony and separation between content blocks.  The root container uses `itemSpacing: 0` to ensure seamless integration between the top and bottom navigation bars and the content area. The content container employs `itemSpacing: 16` for a standard spacing between sections, and individual components utilize internal padding as defined in the design system.

- **Component Behavior**:  The design utilizes existing components where possible, respecting their built-in behaviors and properties. For example, the bottom navigation bar uses the "Bottom navigation" component, while buttons utilize the "Button" component with appropriate size and type variants.  The back button is implemented using a "leading-icon" swap within the "appbar" component, providing a consistent navigational experience.

- **Color Application**:  The color palette strictly adheres to the provided design system.  The primary brand color ("Primary/primary50") is used sparingly for key interactive elements such as the "Contact Seller" button,  while neutral colors ("Neutral/neutral90" and "Neutral/neutral50") are used for the majority of text, ensuring high contrast and readability. The background color is "White/white100" for maximum clarity.

- **Layout Strategy**:  The mobile app adheres to the critical three-container architecture (top navigation, content, bottom navigation), ensuring consistent layout across all screens.  The `layoutMode` is set to "VERTICAL" for the root container and content container to establish a clear top-to-bottom flow.


## Domain Context Integration

- **Industry Patterns**: The design incorporates common e-commerce patterns, such as prominent product images, clear pricing, detailed specifications, seller profiles with ratings, and a prominent call-to-action ("Contact Seller").

- **User Mental Models**: The layout and information architecture are designed to match user expectations in the e-commerce domain.  Key information (price, condition, seller rating) is presented upfront, allowing users to quickly assess the listing.  Specifications and descriptions are provided in an organized manner for easy review.

- **Contextual Adaptations**: The "product card" component, while not explicitly present in the Design System data, is effectively implemented using native elements which adhere to design system specifications. The use of images and text styles ensures that this section looks and feels consistent with the design system even without the "product card" component.


## Design System Compliance

- **Component Verification**: All componentNodeIds are validated against the provided `DESIGN_SYSTEM_DATA`.  Fallback strategies are used where needed.
- **Variant Completeness**:  All variants specified for components have valid values from `DESIGN_SYSTEM_DATA`.
- **Text Property Mapping**: Exact textLayers names from the schema are used.
- **Schema Adherence**: All properties used match the component definitions in `DESIGN_SYSTEM_DATA`.
