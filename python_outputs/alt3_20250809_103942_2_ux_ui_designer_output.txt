**DESIGN RATIONALE:**

1. **Component Inventory Verification:** All component IDs used in this design have been manually verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout prioritizes a vertical flow, placing key information (title, price, condition) prominently at the top.  Product images are presented in a horizontally scrollable layout for easy browsing. Seller information and product details are structured in separate, vertically aligned sections for clarity.  Spacing uses the design system's standard 16px for vertical item spacing within sections and 12px for horizontal spacing in image rows.

3. **User Flow Reasoning:** The user flow is designed to be intuitive and efficient.  The most important information (title, price, and images) is visible first. The user can then scroll down for more details, seller information, and finally, initiate contact.  The "Contact Seller" button provides a clear call to action.

4. **Component Choices:** I chose `10:8492` ("text input") for the contact form fields because it provides a standard input experience, aligned with the design system.  I selected `10:3907` ("Button") for the "Contact Seller" button for consistency with other call-to-action elements across the application.  `147:3743` ("product card") is not used due to image layout requirements differing from the component.

5. **Component Gaps:** The design system lacks a component for displaying multiple product images in a horizontally scrollable carousel. I created a workaround using a `layoutContainer` with `layoutMode: "HORIZONTAL"` and `layoutWrap: "WRAP"` to achieve horizontal scrolling. For the star rating,  I will use multiple instances of the `635:4752` ("icon/star") component to represent a 5 star rating system.

6. **Visual Hierarchy:** Visual hierarchy is established through a combination of typography, size, and position. The product title and price are displayed using larger, bolder text styles for immediate prominence. Product images are visually dominant due to their size and position at the top.  Subsequent sections are progressively smaller and use less prominent typography to maintain visual flow and prevent visual clutter.

7. **Variant Selection Rationale:** For the "Contact Seller" button (`10:3907`), I selected the "Filled," "Medium," and "Green" variants to create a prominent and easily identifiable call to action. The "Default" state reflects standard button appearance.

8. **Native Element Justification:** `native-rectangle` elements are used as image placeholders to display multiple product photos. This was necessary because the design system doesn't provide a component for scrollable image carousels.
