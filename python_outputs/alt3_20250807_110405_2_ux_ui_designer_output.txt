**DESIGN RATIONALE:**

1. **Component Inventory Verification:**  All component IDs have been checked against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout uses a vertical structure for mobile-first design. The app bar occupies the top, followed by a search bar and filter button in a horizontal container.  Product cards are arranged in a wrapped horizontal layout to create a grid, ensuring optimal use of screen space.  Spacing between elements is consistent (16px vertically in primary sections, 12px horizontally in the grid).

3. **User Flow Reasoning:** The user first sees the app bar, providing context and navigation.  The search bar allows for quick product searches, followed by filter options for refinement. The product grid allows users to browse through products.  Clicking on a product card will (implicitly) navigate to the detailed product page.

4. **Component Choices:** I chose the `appbar` (10:5620) component for the top bar because it provides a standard header structure. The `text input` (10:8492) component was used for the search bar as it's the designated input field in the design system. I used the `button` (10:3907) component for the filter button. `product card` (147:3743) component is ideally suited for displaying product information.


5. **Component Gaps:** The design system lacks a dedicated star rating component. A workaround was implemented using multiple `icon/star` (635:4752) components to visually represent the seller rating.

6. **Visual Hierarchy:** The app bar is the most prominent element. The search bar and filter button are secondary elements; the product cards are tertiary.  This is achieved through size and spacing differences; color contrast is kept within the design system's palette.

7. **Variant Selection Rationale:**  For the `button` (10:3907) component, I've selected `"Size": "Medium"`, `"Type": "Filled"`, `"State": "Default"`, and `"Color": "Green"` as these are the best representation of a filter button style.

8. **Native Element Justification:**  No native elements beyond `native-rectangle` (for product images within the card) and `native-text` (for product titles, prices, and potentially seller rating details if star icon use is not possible) were necessary.
