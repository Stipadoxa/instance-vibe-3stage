## DESIGN RATIONALE

1. **Component Inventory Verification:** I have verified all component references against the provided design system data.  All component IDs used are present in `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout employs a vertical structure, mirroring mobile-first design principles. The header contains quick stats cards in a horizontal layout for efficient screen use. The activity feed follows beneath, using a vertical layout for chronological presentation. Action buttons are positioned prominently above the bottom navigation for easy access.  Consistent 16px padding is used for content sections, and 8px spacing between elements within sections.

3. **User Flow Reasoning:**  The dashboard prioritizes at-a-glance information (quick stats) followed by recent activity, enabling quick comprehension of the seller's current status. Prominent action buttons guide users to key tasks. Bottom navigation ensures consistent access to core app features.

4. **Component Choices:** I chose the `product card` (147:3743) component for the quick stats cards because it offers a visually appealing, consistent way to present key metrics. While a custom card could be built, this component reduces development effort and ensures design consistency.

5. **Component Gaps:** The user request requires a list item component for the activity feed, but the design system's `list-item` (10:10214) is not verified and has too many variants making it unsuitable. I used a combination of `native-text` for the activity description and `native-rectangle` with image fills as necessary for a workaround.


6. **Visual Hierarchy:**  Quick stats cards are given visual prominence through larger font sizes (`Headline/Medium`) and distinct background colors.  The activity feed uses a secondary text style and clear spacing for readability. Action buttons use the primary color for high contrast and clear calls to action.

7. **Variant Selection Rationale:** For the action buttons (10:3907), I selected  `Size: Medium`, `Type: Filled`, `State: Default`, `Color: Green` because these options offer optimal visual clarity and consistency with the design system's established button styles.  This was a balance between button size and mobile screen real-estate.

8. **Native Element Justification:** The activity feed required specific formatting of text and images, which could not be achieved using only design system components. `native-text` was used for descriptions and `native-rectangle` with image fills for visual elements in the activity feed.  This ensures the activity feed remains visually appealing and informative.
