**DESIGN RATIONALE:**

1. **Component Inventory Verification:** I have manually checked all component IDs against the provided `DESIGN_SYSTEM_DATA` JSON.  All referenced components exist within the design system.

2. **Layout Decisions:** The layout uses a vertical structure for a clear top-to-bottom flow of notification categories.  Each category is presented as a list item with consistent spacing (16px between items) to ensure readability and visual separation.  Padding is applied to the main content area to create visual breathing room within the viewport.

3. **User Flow Reasoning:** The user can easily scan the list of notification types and toggle each one on or off using the toggle switches.  The descriptive text next to each toggle provides clarity on what each notification type entails.  This direct interaction pattern minimizes cognitive load and allows for quick preference adjustments.

4. **Component Choices:** I chose the `10:9912` (switch) component for its clear on/off visual representation and familiar interaction pattern for toggling notifications.

5. **Component Gaps:** The design system lacks a dedicated "list item with leading toggle switch" component. I've addressed this by using the `10:10214` (list-item) component and positioning the `10:9912` (switch) component as the leading element within each list item.

6. **Visual Hierarchy:** Clear visual hierarchy is established through the vertical arrangement of notification types within the list.  Consistent typography and spacing ensure that each notification type is equally prominent.  The toggle switches provide a direct visual cue to the current status of each notification.

7. **Variant Selection Rationale:** For the `10:9912` (switch) component, I've used the `State: enabled` variant to ensure interactive functionality and the `Selected` variant will dynamically change based on user interaction (true/false).

8. **Native Element Justification:** No native elements were needed because all functional requirements could be met using existing components from the design system.
