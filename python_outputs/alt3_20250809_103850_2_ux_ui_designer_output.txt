**DESIGN RATIONALE:**

1. **Component Inventory Verification:** All component IDs have been checked against the provided `DESIGN_SYSTEM_DATA`.  `10:3907` (Button), `10:8492` (text input), `10:8619` (text area), `10:7805` (Badge), and `147:3743` (product card) are used for their respective functionalities.  No components were used that are not listed in the provided design system.

2. **Layout Decisions:** The layout uses a vertical structure for mobile-first design, prioritizing readability and information hierarchy.  Sections are clearly separated with appropriate padding (`Pattern A`) to avoid a cluttered feel. Image carousels, product details, seller information and action button are arranged logically. The top section holds the product image and title with other information presented progressively downward.

3. **User Flow Reasoning:**  The user journey is designed for smooth and intuitive navigation. The product images are presented first to grab attention, followed by pricing, key features and a description for thorough information gathering. The seller profile with rating builds trust. The Contact Seller button provides a direct call to action.

4. **Component Choices:** The `147:3743` (product card) component was chosen to display product information consistently, rather than using individual text and image components, ensuring alignment with the design system.

5. **Component Gaps:** The design system lacks a suitable component for an image carousel, and a star rating system.  To address these gaps, `native-rectangle` components with image fills are used for the image carousel, and multiple `635:4752` (icon/star) components are used to represent the seller rating.

6. **Visual Hierarchy:** Visual hierarchy is established through a combination of layout position, typography, and color. The product title uses a larger text size and prominent positioning compared to other elements.  The color palette is consistent and adheres to the design system.


7. **Variant Selection Rationale:**  For the button (`10:3907`), the "Filled", "Large", "Default", and "Green" variants were selected to promote a clear and prominent call to action.  Other component variants are selected based on context and default states.

8. **Native Element Justification:**  `native-rectangle` elements were used to display the multiple product images due to the lack of a dedicated image carousel component in the design system.  Multiple `635:4752` (icon/star) components were used to create a star rating for the seller since a star rating component is not present.
