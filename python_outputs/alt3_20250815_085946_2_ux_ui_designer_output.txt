## SECTION 1: DESIGN RATIONALE

# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish clear visual hierarchy.  `Headline/Large` is used for the main product title, providing prominence.  `Title/Medium` styles section headers (like "Seller Profile" and "Specifications"), ensuring readability and logical grouping.  `Body/Large` is used for the product description to ensure comfortable reading, while `Body/Medium` styles the specifications, creating a clear distinction between the main content and supporting details.  `Caption/Small` is used for timestamps ("Last updated: 2 hours ago").

- **Spacing System**:  The `itemSpacing` token is used consistently throughout the layout.  The root container uses `itemSpacing: 0` to ensure seamless integration between the top and bottom navigation bars and the content area. Within the content container, `itemSpacing: 16` provides standard spacing between content blocks. Padding values are applied consistently within the main content container (`paddingTop/Bottom: 16`, `paddingLeft/Right: 16`), creating visual breathing room without compromising screen real estate.

- **Component Behavior**: The `appbar` component (`10:5620`) is used for the top navigation,  providing a consistent header style across the app.  The `Bottom navigation` component (`10:4724`) is used for bottom navigation, offering readily available core app functions. The "Contact Seller" button uses the `Button` component (`10:3907`), with the `Size` variant set to "Large" and `Type` set to "Filled" to highlight its primary action status. The "Report Listing" button utilizes the same `Button` component but with the `Size` variant set to "Small" and `Type` set to "Text" to reduce visual prominence.


- **Color Application**:  The color palette is rigorously implemented using the design system's naming conventions.  The main background uses `White/white100`.  Primary text uses `Neutral/neutral90`, while secondary text utilizes `Neutral/neutral50`. The primary action button ("Contact Seller") uses `Primary/primary50` for its background and `White/white100` for its text color. Secondary elements such as labels use `Neutral/neutral50`.  This ensures consistency and brand recognition.


- **Layout Strategy**: The design strictly adheres to the three-container mobile architecture: top navigation, content area, and bottom navigation.  This ensures consistent app structure and optimal use of screen space across all views. The `layoutMode` is set to `VERTICAL` for the root and content containers, supporting a top-to-bottom information flow.  Horizontal alignment is used within the content container as needed to organize related elements.


## Domain Context Integration

- **Industry Patterns**: The design incorporates common e-commerce patterns such as product image carousels, detailed specifications, pricing information, user reviews, and seller profiles. This familiar structure reduces cognitive load for the user.

- **User Mental Models**:  The layout prioritizes key information (price, condition, images) prominently, aligning with user expectations when reviewing product listings. Important actions are clearly placed, and supporting information is organized logically, facilitating efficient decision-making.

- **Contextual Adaptations**: The absence of a "cart" in bottom navigation reflects the unique context of a used-goods marketplace where direct seller contact is crucial. The design prioritizes this interaction over traditional e-commerce cart functionalities.


## Design System Compliance

- **Component Verification**: All componentNodeIds are verified against the provided `DESIGN_SYSTEM_DATA`.
- **Variant Completeness**: All components utilize appropriate variants. For instance, `Button` component variants accurately reflect primary vs. secondary actions.
- **Text Property Mapping**: Text content maps directly to the correct `textLayers` within each component.
- **Schema Adherence**: Component properties adhere strictly to the defined schema within `DESIGN_SYSTEM_DATA`.
