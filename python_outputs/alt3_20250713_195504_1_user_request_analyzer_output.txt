Domain & Context
Domain: Business/SaaS domain with user account management and settings configurations.  Specific context is a mobile application likely related to auctions or a similar marketplace.
User Type: Mobile app users managing their account preferences and settings.
Platform Impact: Mobile users need a concise, easily navigable settings page with clear actions and touch-friendly controls. Limited screen real estate necessitates efficient information architecture.

User Requirements
Primary Goals: Access and modify personal information, manage app preferences, control account security, and logout securely.
Secondary Goals: Understand the meaning of badges or status indicators, easily access help or support documentation where needed.
User Constraints: Small screen size, touch-only interaction, potential for interruptions, limited attention span.
Ready for handoff if:  All settings options from the reference image are included using the provided design system components,  navigation is intuitive, and touch targets meet minimum accessibility standards.


Content Requirements

Supporting:  "What is it?" explanation for unclear settings options.
Contextual:  Success/failure messages for settings modifications, loading indicators during processing.

* Layout:  A single-column list view, similar to the reference image, with clear separators between settings items.  The layout must adhere to the provided design system.
* Components:  List items should utilize the provided design system components for settings options. Each option should use the appropriate component type (e.g., toggle switch for on/off settings, text input for editable fields, etc.).

Screen Requirements

Secondary Actions:  Access help documentation or explanations for setting options.
