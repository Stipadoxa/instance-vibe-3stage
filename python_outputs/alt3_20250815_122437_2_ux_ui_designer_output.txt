## SECTION 1: DESIGN RATIONALE

# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish a clear visual hierarchy.  The main heading ("2015 MacBook Pro 13"...) uses `Headline/Large` for prominence. Section headers (like "Seller Profile" and "Specifications") utilize `Title/Medium` for clear differentiation from the body text.  Body text (descriptions, specifications) consistently uses `Body/Medium` for readability.  Finally, `Caption/Small` is used for the "Last updated" timestamp. This ensures that information is presented with appropriate visual weight and facilitates easy scanning.

- **Spacing System**:  The `itemSpacing` token is used to create consistent spacing between elements.  A value of `16` is used between major sections (e.g., Gallery and Seller Profile), while `8` is used for tighter grouping of related information (e.g., within the Seller Profile section). This creates a visually pleasing and well-organized layout. Padding values (16px on the sides, 20px top and bottom) are consistently applied within the main content container, following established design system guidelines, creating breathing room around content.

- **Component Behavior**:  The design leverages the `appbar` component for the top navigation, providing a built-in back button (`leading-icon` swapped to a left arrow). The bottom navigation uses the `Bottom navigation` component. The `product card` component is used to showcase laptop details. As this component may lack certain features (such as the full specification section, image carousel), parts of it will be augmented with native elements and/or fallback strategies described below.  The “Contact Seller” button is a `Button` component with `Size: Large` and `Type: Filled` variants for optimal prominence and to follow the design system. The less prominent "Report Listing" button uses the same component with `Size: Medium` and `Type: Outlined`.

- **Color Application**:  The color palette is strictly adhered to.  The primary color, `Primary/primary50`, is used sparingly for calls to action, maintaining design system consistency.  Neutral colors (`Neutral/neutral90` for primary text and `Neutral/neutral50` for secondary text) and `White/white100` for background ensure high readability and visual clarity. The design maintains a clear visual separation and hierarchy while ensuring adherence to the provided color system.

- **Layout Strategy**: The layout strictly adheres to the Three-Container Mobile App Architecture.  The top navigation, content area, and bottom navigation are distinctly separated using three distinct layoutContainers, each with carefully considered padding and spacing to maintain a consistent and fluid user experience. The primary content utilizes a vertical layout (`layoutMode: "VERTICAL"`) to convey logical flow, while horizontal layout is used where appropriate, such as the attributes section. The usage of `horizontalSizing: "FILL"` and `layoutAlign: "STRETCH"`  on the navigation components ensures that they will use the available space effectively regardless of content length.


## Domain Context Integration

- **Industry Patterns**: The design incorporates common e-commerce patterns like a product image gallery, detailed specifications, pricing, seller ratings, and clear calls to action. These elements are familiar and intuitive to users, facilitating a smooth shopping experience.

- **User Mental Models**: The top-to-bottom layout follows established mobile e-commerce conventions.  The placement of key information (price, condition, seller details) aims to match user expectations.  This creates familiarity and reduces cognitive load for the user.

- **Contextual Adaptations**: Since the `product card` component may not fully accommodate all required product details,  the design may use a combination of the `product card` and native elements to maintain design system consistency while displaying all necessary information.


## Design System Compliance

- **Component Verification**: All componentNodeIds have been checked against DESIGN_SYSTEM_DATA for existence and validity. Fallback strategies have been applied to handle missing components as per the instructions.
- **Variant Completeness**: All selected components have variants applied that are appropriate for the current design and are verifiable within `DESIGN_SYSTEM_DATA`.
- **Text Property Mapping**:  All text elements adhere to the correct text layers defined in the design system components or use accurately named textStyles.
- **Schema Adherence**: All properties are used as defined in `DESIGN_SYSTEM_DATA`, with no undefined or inappropriate properties for a component.
