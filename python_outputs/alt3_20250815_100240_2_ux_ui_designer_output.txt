# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided design system's typography to establish a clear visual hierarchy.  "Headline/Large" is used for the main product title ("2015 MacBook Pro 13" - Excellent Condition"), providing prominence.  "Title/Medium" styles section headers (e.g., "Specifications"), and "Body/Large" is used for the main product description. "Body/Medium" styles the individual specifications, ensuring readability and clear distinction between different levels of information.  "Label/Medium" styles labels like "Processor:", "RAM:", etc., providing a clear visual cue for these descriptive elements.

- **Spacing System**: The three-container mobile architecture ensures consistent spacing and edge-to-edge design.  The content area utilizes `itemSpacing: 16` for primary spacing between content blocks (image carousel, price, condition, seller profile, description, specifications).  Internal padding within these blocks is determined by individual component padding.

- **Component Behavior**: The `appbar` component (id: `10:5620`) is used for the top navigation, with the `leading-icon` swapped to a left-pointing arrow ("back" icon). The `Bottom navigation` component (id: `10:4724`) is used for the bottom navigation bar, displaying icons for Home, Search, Messages, and Profile.  The "Contact Seller" button is a "Button" component (id: `10:3907`) with `Size: Large` and `Color: Green` for prominence. The "View Seller Profile" link uses native text with the "Body/Medium" text style to create a visual distinction.

- **Color Application**: The design system's color palette is consistently applied. The background uses `White/white100`. Primary text uses `Neutral/neutral90`, while secondary text uses `Neutral/neutral50`. The primary action button ("Contact Seller") utilizes `Primary/primary50` for the background color.

- **Layout Strategy**: The design strictly adheres to the three-container mobile architecture (Top Navigation, Content Container, Bottom Navigation), ensuring a consistent and predictable layout across all screens. The vertical layout (`layoutMode: "VERTICAL"`) is used for the main content area, placing information in a logical order that matches user expectations.  Horizontal containers (`layoutMode: "HORIZONTAL"`) are potentially used inside the content area, for instance, to group specifications for better readability.


## Domain Context Integration

- **Industry Patterns**: The design mirrors standard e-commerce product detail pages, featuring prominent product imagery, pricing, seller information, and clear calls to action.

- **User Mental Models**: The layout prioritizes information based on user importance (e.g., image carousel is first, followed by price and condition). This matches typical user scanning behavior on e-commerce product pages.

- **Contextual Adaptations**: The "product card" component (id: `147:3743`) could potentially be adapted, although a custom solution with native components is implemented to accommodate the unique content requirements of this specific product detail page.


## Design System Compliance

- **Component Verification**: All componentNodeIds have been checked and confirmed to exist within the DESIGN_SYSTEM_DATA.

- **Variant Completeness**: Where applicable, all required variants for chosen components (e.g., "Button" component's size, style, and state) have been specified.

- **Text Property Mapping**: All text elements are mapped to the correct `textLayers` within the selected components where possible. Native text is styled using the design system's text styles.

- **Schema Adherence**: All components' properties adhere strictly to the provided schema in terms of their data types and allowed values.
