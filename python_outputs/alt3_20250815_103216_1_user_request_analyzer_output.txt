## STAGE 1: UNDERSTAND THE REQUEST

1. **Domain:** E-commerce
2. **Platform:** Mobile
3. **User Goal:** The user wants to view the details of a used laptop listing on a marketplace app before potentially purchasing it.


## STAGE 2: GENERATE CORE CONTENT

**Main Screen Purpose:** To display comprehensive information about a used laptop listing, allowing the user to decide whether to contact the seller.

**Required Information & Actions:**

* **Heading:** 2015 MacBook Pro 13-inch - Excellent Condition

* **Key Information Blocks:**
    * **Gallery:** (Carousel of at least 4 high-quality images of the laptop from different angles, showing its condition clearly.  Includes a primary image displayed prominently).
    * **Price:** $450
    * **Condition:** Excellent - Minor scratches on the casing, fully functional.
    * **Seller Profile:**  (Small profile image)  John Doe (5.0 stars, 12 reviews)
    * **Description:** "Lightly used 2015 MacBook Pro 13-inch.  Comes with original charger. Battery life excellent.  Upgraded to 16GB RAM.  Minor cosmetic scratches on the casing (see photos)."
    * **Specifications:**
        * Processor: 2.7 GHz Intel Core i5
        * RAM: 16GB
        * Storage: 512GB SSD
        * Operating System: macOS Monterey


* **Primary & Secondary Actions:**
    * **Primary:** Contact Seller (Button)
    * **Secondary:**  View Seller Profile (Button)


* **Required Status Indicators:**
    * Last updated: 1 hour ago (Dynamically updates)


## STAGE 3: ADD NAVIGATION

**Top Navigation:**

* Back (Icon: Left-pointing arrow)

**Bottom Navigation:**

* Home (Icon: House)
* Search (Icon: Magnifying glass)
* Messages (Icon: Speech bubble)
* Profile (Icon: Person)

(Note: This bottom navigation is a common pattern for marketplace apps, assumed to be consistent across the app. If the user requests otherwise, this should be adjusted accordingly.)
