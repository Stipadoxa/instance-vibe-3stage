## SECTION 1: DESIGN RATIONALE

# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish a clear visual hierarchy.  "Headline/Large" is used for the main product title, providing prominence. "Title/Medium" styles section headers (like "Seller Profile" and "Specifications"), while "Body/Large" and "Body/Medium" are used for the product description and specification details respectively.  "Caption/Small" is used for less important information like the listing date. This ensures readability and guides the user's attention.

- **Spacing System**:  Consistent spacing is maintained using the design system's `itemSpacing` tokens.  `itemSpacing: 16` is used as the default spacing between major sections (e.g., Product Images and Seller Profile) to create visual breathing room.  Within sections, `itemSpacing: 8` or `itemSpacing: 12` is applied where appropriate for tighter grouping of related information.  Padding is used consistently to create a balanced and comfortable layout. For example, `paddingLeft/Right: 16` is applied to main content sections.

- **Component Behavior**: The design utilizes the `list-item` component for the specifications table, leveraging its "2-line" variant to accommodate both the attribute name and value. The "Contact Seller" button uses the `Button` component with "Large" size and "Primary" color variant. The "Report Listing" button uses the "Medium" size and "Secondary" color variant to signify it's a secondary action.  The bottom navigation utilizes the "Bottom navigation" component, ensuring a consistent experience across the app.

- **Color Application**: The color palette is carefully selected to enhance readability and align with design system guidelines.  "White/white100" is used as the primary background, while "Neutral/neutral90" is selected for primary text, ensuring sufficient contrast. "Primary/primary50" is used for CTAs (Call to Action) buttons and "Secondary/secondary50" is used for secondary actions, reflecting the design system's color hierarchy.

- **Layout Strategy**:  A vertical layout is adopted to follow the natural reading pattern.  Sections are arranged logically, starting with prominent visual elements. Each section uses a `layoutContainer` component with flexible sizing adjusted for optimal display on a mobile screen. The `primaryAxisSizingMode` is set to "AUTO" to allow the section height to dynamically adjust to the content.

## Domain Context Integration

- **Industry Patterns**: The design adheres to e-commerce conventions by prominently displaying product images, price, seller information, and clear call-to-action buttons.  The specifications are presented in a structured table format for easy understanding. The user review and seller rating information is given due prominence.

- **User Mental Models**: The layout is organized to match user expectations in an e-commerce context. The top navigation provides a clear “Back” action for a smooth user journey. The bottom navigation facilitates intuitive navigation. Key information is prioritized and presented visually appealingly.

- **Contextual Adaptations**: The design is tailored to the specific use case. A carousel is chosen for images to allow the user to view all images conveniently. The description field is implemented with native elements to support a wider variation in user supplied content.  The seller's rating is directly presented instead of relying on a user's assumption of a rating system.


## Design System Compliance

- **Component Verification**: All componentNodeIds used were validated against the provided `DESIGN_SYSTEM_DATA`.
- **Variant Completeness**:  All selected components have appropriate variants applied, based on the contextual needs.
- **Text Property Mapping**:  Text content is mapped to the appropriate `textLayers` of the chosen components.
- **Schema Adherence**:  All properties and values adhere to the component definitions in `DESIGN_SYSTEM_DATA`.
