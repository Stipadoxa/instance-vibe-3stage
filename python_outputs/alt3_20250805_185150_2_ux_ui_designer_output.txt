**DESIGN RATIONALE:**

1. **Component Inventory Verification:**  All component IDs used in the design specification below have been verified against the provided `DESIGN_SYSTEM_DATA`.

2. **Layout Decisions:** The layout uses a vertical structure. The app bar occupies the top, followed by the search bar and filter button in a horizontal container. The product grid uses a horizontal container with `layoutWrap: "WRAP"` to create a responsive grid of product cards.  Each product card has a fixed height to maintain consistency, allowing for varying width based on the content. Consistent spacing of 16px is used between major sections and 8px between items within the product grid.  Full-width elements, such as the hero image in the product card utilize `"horizontalSizing": "FILL"`.

3. **User Flow Reasoning:** The user first sees the search bar and filter options at the top for easy access.  The product grid provides a clear visual hierarchy with the product cards, encouraging browsing. The product cards themselves present key information concisely (image, title, price, rating) to aid quick decision-making.

4. **Component Choices:**  For the search bar, I chose the `10:12086` ("search-field") component instead of creating a custom input field with native elements. It offers native search functionality and design system compliance.  The filter button uses the `10:3907` ("Button") component for consistency. The product cards are created using the `147:3743` ("product card") component, chosen over manually creating card layouts with native elements because it offers pre-built styling and maintainability.

5. **Component Gaps:** The design system lacks a built-in rating component. To display seller ratings, I have used five instances of the `635:4752` ("icon/star") component as a workaround. Each star can be controlled with variants to indicate "full", "half", or "empty" states, allowing for the creation of dynamic star ratings.

6. **Visual Hierarchy:** The app bar establishes primary importance. The search bar and filter are secondary, and the product grid is the main content area.  Within the product grid, larger title text and image sizes create prominence.   Color contrasts and spacing further enhance visual flow.

7. **Variant Selection Rationale:** For the "search-field" component (`10:12086`), the "default" state is used as the initial state.  For the "Button" component (`10:3907`), the "Medium", "Filled", "Default", and "Green" variants are selected for better visual integration.  The "product card" component (`147:3743`) uses the "grid" and "normal" view and type variants for the intended layout.

8. **Native Element Justification:** To create the product image placeholders within the product cards, I used `native-rectangle` elements with `fill: {type: "IMAGE"}` to ensure that product images are displayed. This approach avoids forbidden `native-image` elements, whilst providing the desired image functionality.
