## SECTION 1: DESIGN RATIONALE

# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided typography system to establish clear visual hierarchy.  "Headline/Large" is used for the main product title, providing prominence. Section headers like "Seller Profile" and "Specifications" utilize "Title/Medium" for clear distinction. Body text for the description and table values uses "Body/Medium" for readability.  "Caption/Small" is reserved for the "Last updated" timestamp. This consistent application of text styles ensures easy scanning and comprehension of information.

- **Spacing System**:  The three-container mobile architecture (top nav, content, bottom nav) ensures a consistent layout. `itemSpacing` of 16 is used between major content blocks within the central content container to provide visual breathing room.  Internal padding within components is respected and not overridden unless explicitly needed for layout harmony.  Padding within the central content container is set to 16px on the sides and 20px top/bottom for a comfortable reading experience.

- **Component Behavior**: The design utilizes the `appbar` component for the top navigation, incorporating the "back" and "share" icons. The bottom navigation uses the `Bottom navigation` component.  The "Contact Seller" button is implemented as a `Button` component with `Size: Large` and `Type: Filled` for high visibility and emphasis, while the "Add to Wishlist" button uses `Size: Medium` and `Type: Outlined` as a secondary action.  The product image carousel is created using the `IMAGES` component, with a fallback for cases where no images are available.

- **Color Application**: The color palette is chosen to enhance readability and convey a sense of trust and professionalism.  The primary color (`Primary/primary50`) is used sparingly, mainly for the price and primary CTA button.  Secondary colors are avoided, while neutral colors (`Neutral/neutral90` for primary text, `Neutral/neutral50` for secondary text) ensure high contrast and readability. The background uses `White/white100` for maximum readability.

- **Layout Strategy**: The `layoutMode` is predominantly "VERTICAL", ensuring a natural reading flow from top to bottom.  The "Specifications" table is structured as a separate layout container. The root layout leverages the three-container mobile app pattern with `itemSpacing: 0` for seamless transitions between the navigation bars and content.


## Domain Context Integration

- **Industry Patterns**: The design adheres to e-commerce best practices by prominently displaying the product title, price, condition, images, and seller information.  Clear calls to action encourage user engagement.

- **User Mental Models**: The layout prioritizes information in a way that aligns with the user's likely expectations when viewing a product listing: images first, then price, description, seller info, and finally actions.

- **Contextual Adaptations**:  The design includes fallbacks for missing data (e.g., "No ratings yet" for seller rating) and empty image carousels ("No images yet").


## Design System Compliance

- **Component Verification**: All componentNodeIds have been checked against the `DESIGN_SYSTEM_DATA` and confirmed to exist.

- **Variant Completeness**: All component variants have been selected according to the design rationale and context.  Fallbacks are used where no exact matches in the design system were found.

- **Text Property Mapping**:  Text content is mapped to the appropriate `textLayers` names defined within the components where possible.

- **Schema Adherence**: All properties used adhere to the defined component schemas in `DESIGN_SYSTEM_DATA`.
