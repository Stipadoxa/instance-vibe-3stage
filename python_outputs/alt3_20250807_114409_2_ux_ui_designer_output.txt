**DESIGN RATIONALE:**

1. **Component Inventory Verification:** All component IDs have been manually checked against the provided `DESIGN_SYSTEM_DATA` JSON.  No invented component IDs were used.

2. **Layout Decisions:** The layout uses a vertical, mobile-first approach. The screen is divided into distinct sections: Hero Image Section (multiple images in a scrollable container), Product Details Section (title, price, condition), Seller Profile Section (avatar, rating), Description Section, and Action Section (Contact Seller button).  Spacing between sections is 16px, and internal padding within sections is 16px for consistency and readability.

3. **User Flow Reasoning:** The user flow prioritizes information hierarchy, placing the most crucial details (images, title, price) at the top.  The description follows, giving users context.  The seller profile and contact button are readily accessible. The visual hierarchy guides the user through the information.

4. **Component Choices:**  The `product card` component (ID: 147:3743) was considered for its potential to encapsulate product details, but  it doesn't meet the multi-image gallery requirement and seller rating. This necessitated a customized approach.

5. **Component Gaps:** The design system lacks a dedicated image gallery component and a star rating component.  For the image gallery, I use a `layoutContainer` with `layoutMode: "HORIZONTAL"` and `layoutWrap: "WRAP"` to create a horizontally scrolling gallery.  For the star rating, I will use multiple `native-circle` components to represent stars.

6. **Visual Hierarchy:** Visual hierarchy is established through size, position, and color. The hero images are prominent at the top.  Headings use larger text sizes, and body text is smaller.  Primary colors are used sparingly for emphasis on key calls to action.

7. **Variant Selection Rationale:**  For the "Contact Seller" button (`10:3907`), I use the `Size: "Medium"`, `Type: "Filled"`, `State: "Default"`, and `Color: "Green"` variants to ensure consistency and clear visual cue.

8. **Native Element Justification:** The lack of a dedicated image gallery and star rating component necessitated the use of `native-rectangle` (for images, using image fills) and `native-circle` (for stars in the rating system).
