# DESIGN RATIONALE

## Component Intelligence Applied

- **Typography Hierarchy**: The design leverages the provided headline styles for primary titles ("Headline/Large" for the main product title), section headers ("Headline/Medium" for sections like "Specifications"), and subsection headers ("Headline/Small" where needed within specification subsections). Body text uses "Body/Large" for the description and "Body/Medium" for specifications.  This clear hierarchy ensures readability and visual prominence for key information.

- **Spacing System**:  `itemSpacing` tokens (8, 12, 16) are used consistently to create visual groupings and separation between elements.  Padding (16px) is applied to the main content area to ensure comfortable spacing around the content within the screen.  The three-container architecture ensures top and bottom navigation bars are flush with screen edges.

- **Component Behavior**: The `appbar` component is used for top navigation, showing a back button and the title "Product Details". The `bottom navigation` component is utilized for bottom navigation.  Fallback strategies are applied where necessary.  For example, given the absence of a dedicated 'product-card' component, a 'list-item' is used with appropriate variants.

- **Color Application**: The color system is strictly adhered to.  Primary colors (#00b53f variations) are used for the main CTA button ("Contact Seller"). Neutral colors (#304049 and variations) are used for text and backgrounds, ensuring high contrast and readability.

- **Layout Strategy**:  The design uses the mandatory three-container architecture for mobile apps (Top Navigation, Content Container, Bottom Navigation) ensuring a consistent user experience.  Vertical layout is used for the main content flow,  and horizontal layouts are used within sections to display related information concisely.


## Domain Context Integration

- **Industry Patterns**: The design incorporates common e-commerce patterns such as product image carousels, detailed specifications, pricing information, seller profiles with ratings, and clear CTAs. The user flow mirrors typical product detail page interactions.

- **User Mental Models**: The layout is organized intuitively, placing key information (price, condition, images) prominently above the description and specifications.  This follows user expectations for product detail pages and allows for quick scanning of essential information.

- **Contextual Adaptations**:  The "list-item" component is used as a fallback for the missing "product-card" component, demonstrating adaptability to design system constraints. The app bar title is concise, following the character limit to allow for comfortable visibility in a mobile app bar.


## Design System Compliance

- **Component Verification**: All componentNodeIds are verified against the DESIGN_SYSTEM_DATA and are real component IDs.
- **Variant Completeness**: All specified variants and their values are verified against the DESIGN_SYSTEM_DATA.
- **Text Property Mapping**: Text properties are mapped accurately to the textLayers within the selected components and native-text elements use appropriate text styles.
- **Schema Adherence**: All components and their properties strictly adhere to the schemas outlined in the provided DESIGN_SYSTEM_DATA.
